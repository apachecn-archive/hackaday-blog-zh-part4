# 击败 Wii Mini，互联网就在你的身后

> 原文:[https://hack aday . com/2019/07/12/击败 Wii-mini-as-the-internet-watches-over-your-should/](https://hackaday.com/2019/07/12/defeating-the-wii-mini-as-the-internet-watches-over-your-shoulder/)

在被直播观看的压力下工作，【deadly foez】[与所谓的无法破解的 Wii Mini 对抗，并展示了前所未有的结果](https://gbatemp.net/entry/deadlyfoez-vs-wii-mini-tonight-10-pm-est-part-one.15873/)，同时记录了他的过程，供其他人跟随。我们挖掘了这些内容来寻找这个过程的精华，下面的链接包括这些时刻的时间戳。

Wii Mini 是任天堂最畅销游戏机的低价版本，在接近生命周期结束时出售，删除了一些功能，如 GameCube 向后兼容和 SD 卡支持。与此同时，为了阻止困扰其姐姐任天堂的越狱，任天堂制作了 NAND 内存(系统存储在那里)进行加密，并与每台设备的好莱坞 GPU 芯片相匹配。这挫败了为了获得对硬件的访问而修改存储器的方法。

无论如何，这并没有阻止[DeadlyFoez]的尝试，在一个已经被黑了的 Wii 捐助者的帮助下，计划出他需要的步骤来实现一个被黑的迷你单元。在扔掉两个系统的 NAND 并进一步探索 Wii Mini 硬件后，他发现了一些惊喜。板上有测试点，允许 GameCube 控制器与它一起使用。主板上也有 SD 卡连接，但 Mini 的系统软件中取消了这种支持。

然而，最有趣的部分出现在后面:通过在最初的 Wii 和 Wii Mini 之间同时交换 NAND 和 GPU 芯片，[DeadlyFoez]设法将两个不同的系统放在一起。第一个是[一个带有 Mini 芯片的原始 Wii 板](https://youtu.be/rBpNxcKhLoE?t=164)声称是“第一个运行自制软件的 Wii Mini”。第二，填补等式的另一边，用硬件和软件来[给 Wii Mini](https://youtu.be/GuttbwSdR1A?t=378) 添加 SD 卡和 GameCube 控制器端口。

为了将任天堂硬件改造成非正统版本，BGA 返工的过程实际上在几年前就已经完成了，当时有人通过将两个独立控制台的部件拼凑在一起，制作了一个非官方的美国地区非 XL 新 3DS。
