# 货物养殖和埋藏的宝藏

> 原文:[https://hack aday . com/2022/12/03/货藏宝/](https://hackaday.com/2022/12/03/cargo-culting-and-buried-treasure/)

我不知道这些故事有多真实，但传说在第二次世界大战期间，当物资被空投到一些美拉尼西亚群岛时，一些当地人开始复制跑道标志，以进一步取悦空投物资的“神”。据报道，他们认为制造简易机场*会导致*满载飞机到访。理查德·费曼后来[把这变成了一个关于科学理论的隐喻](https://en.wikipedia.org/wiki/Cargo_cult_science)——如果你不深刻理解你在做什么，你可能在欺骗自己。

我想对科技货物崇拜者宽容一点。因为我们周围的世界是非常复杂的，我们经常只是按照事物的本来面目去看待它们，而不是深入理解它们，因为你能进入的领域太多了。

一个不懂 i386 机器语言的人是不是在通过一份 web 后端开发人员的工作来培养自己？大概不会。但是从汇编语言程序员的角度来看，我们任何用编译或解释编程语言编写的人都是在培养编码能力。你不需要懂手机就能打电话回家，但是你真的能说你了解一个人如何工作的一切吗？还是你只是走过场？

因此，虽然依赖隐喻和“嗯，上次成功了”是完全正常的，但我认为注意到你对货物的崇拜也是健康的。这也应该是一个警告标志，或者至少是一面提醒自己这里可能有龙的旗帜。或者可能只是一个被埋没的学习机会，X 标志着深入挖掘可能会有成效的地方。
