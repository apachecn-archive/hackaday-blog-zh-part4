# 塞尔达传说:反编译

> 原文：<https://hackaday.com/2021/12/24/the-legend-of-zelda-decompiled/>

出于多种原因，保持程序源代码的封闭性在这里是不被允许的。封闭源代码不太安全，也不太容易定制，但不幸的是，我们无法很快说服每个人开放源代码的优点。另一方面，为了更好地理解那些代码，反编译那些源代码仍然被锁在门后的程序是可能的，最近一个更令人印象深刻的例子是[这个项目完全反编译了时间](https://github.com/zeldaret/oot)的陶笛。

要开始编写这个项目的代码，只需要克隆 Git 库，然后使用一组特定的软件工具(取决于用户的操作系统)从源代码编译 ROM。然而，从那里开始，世界就是你装满卢比的罐子。就像我们从其他反编译游戏中看到的那样，可以对原始游戏进行任何数量的增强，包括提高帧速率，改善图形，或者添加原本不会出现的华丽元素。

这个项目的创建者确实指出，这仍然是一项正在进行的工作，因为 18 个版本中只有一个已经完成，但事实上，他们能够反编译的源代码在重新编译时构建了一个完全可用的游戏，这说明了它已经走了多远。我们已经看到之前的其他游戏使用了[类似的过程，这也有助于说明当从它们的源代码重写旧游戏时有多少改进是可能的。](https://hackaday.com/2019/06/08/hail-to-the-king-baby-reverse-engineering-duke/)
