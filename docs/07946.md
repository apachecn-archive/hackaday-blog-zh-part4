# OpenFluid Warmer 旨在将医疗设备送到需要的地方

> 原文:[https://hack aday . com/2020/10/08/open fluid-warmer-aims-to-get-medical-equipment-where-its-need/](https://hackaday.com/2020/10/08/openfluid-warmer-aims-to-get-medical-equipment-where-its-needed/)

静脉注射液是现代救生医学的重要组成部分。取决于所讨论的流体，它们必须经常在低温下储存，然而，为了输送给患者，将它们加热到大约 38 度以避免引起体温过低是有益的。为了实现这一点，使用了 IV 液体加温器，但是这些在世界各地都不容易获得。为了帮助纠正这个缺点，[【John ops ahl】启动了 OpenFluidWarmer 项目。](https://hackaday.io/project/173308)

该项目的目标是生产一种安全、可靠且易于复制的静脉输液加温器。使用的材料必须便宜且易于获得，理想情况下应易于替代，以最大限度地提高设计能力，在任何需要的地方建造。该项目的名称是对其开放设计的认可——该项目的目标是向那些没有医疗设备的人提供医疗设备，将设计保密没有什么好处。

开发继续稳步进行，优化加热器性能、固件，甚至组装所需的工具[都记录在构建日志中。](https://hackaday.io/project/173308-openfluidwarmer#menu-logs)这个项目让人回想起今年早些时候[为新冠肺炎患者创造开源呼吸机的争夺战。](https://hackaday.com/2020/03/12/ultimate-medical-hackathon-how-fast-can-we-design-and-deploy-an-open-source-ventilator/)归根结底，这是为了将医疗硬件送到最需要的地方，我们对[John]在这一领域的努力表示赞赏！休息后的视频。
