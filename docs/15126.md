# 借助 Python 生成立体声调频信号

> 原文:[https://hack aday . com/2022/10/19/generating-stereo-FM-signals-thanks-to-python/](https://hackaday.com/2022/10/19/generating-stereo-fm-signals-thanks-to-python/)

对调幅广播的工作原理很容易理解，标准调频广播只是稍微复杂一点。不过，立体声调频可能会有点跑偏——弄清楚他们是如何将两个独立的音轨压缩到一个无线电信号中的，有点让人头疼。在这种情况下，用任意波形发生器和一点 Python 语言模拟一个立体声 FM 信号，可能会有助于你理解这个概念。

提醒你一下，波罗的海实验室的塞巴斯蒂安并不是不熟悉多路调频理论。据他解释，他的目标是产生一个有效的立体声调频信号，每个通道都有不同的纯音，左边 700 赫兹，右边 2200 赫兹。幸运的是，[Sebastian]有一个不错的 AWG，Siglent SDG1032X，它有一个以太网连接，可以用来远程控制它和 PyVISA，PyVISA 是一个 Python 包，用于使用虚拟仪器软件架构协议控制仪器。

这个项目的核心，以及真正有助于将多个音频信号放入一个 FM 信号的概念，在于生成组件的 Python 代码。[Sebastian]很好地解释了他如何通过编程产生和差信号以及 19 kHz 导频音，并将它们放入一个波形中。该程序的输出用于产生一系列值，这些值被发送到任意波形发生器，该发生器输出所需的 FM 信号。查看频谱分析仪的输出，两个音频音调清晰可见，衰减的导频音和稍高一些的其他尖峰也是如此。

只要在系统中增加一根天线，你就拥有了世界上最乏味的调频广播电台——但至少它是立体声的。或者，如果您想了解调频广播的起源，[我们也为您准备了一些东西](https://hackaday.com/2018/07/24/edwin-armstrongs-battle-for-fm-radio/)。
