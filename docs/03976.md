# 小型电脑迎来 Qt

> 原文:[https://hack aday . com/2019/08/22/Qt-arrives-for-small-computers/](https://hackaday.com/2019/08/22/qt-arrives-for-small-computers/)

曾经有一段时间，编写嵌入式系统意味着永远不必处理图形用户界面，也不必花费很长时间试图释放十几个字节的 ROM 来添加一个功能。如今，一个嵌入式系统可能会有一个屏幕和一个巨大的内存，即使是在不到十年前的个人电脑上。长期以来，Qt 一直是在桌面平台上构建软件的流行选择，尽管没有那么流行，但它甚至已经在手机上运行了一段时间。现在，MCU 的 [Qt 显然瞄准了每个人都试图占领的物联网市场。你可以在下面看到新产品的炫目视频。](https://www.qt.io/qt-for-mcu)

我们通常喜欢 Qt，最近的趋势是转向一种叫做 QML 的类似 HTML 的标记语言，而不是直接操作小部件。我们认为这是一件好事。然而，Qt 不仅仅用于用户界面。它以简单明了的方式提供广泛的服务

最大的吸引力是该系统是便携式的。你主要在 Linux 上看到 Qt 程序，但是它们也可以在很多其他地方工作，包括 Windows 和很多手机。这将继续把 Qt 推向尽可能多的地方。你可以争辩说 Java 做到了这一点，但是一个真正的资源友好和无争议的 Java 一直是难以捉摸的。即使在大型计算机上，人们也经常开玩笑说 Java 是“编写一次，到处调试”——这是对“编写一次，到处运行”口号的一种戏谑。

回到 2016 年，我们看了一下[Qt creator](https://hackaday.com/2016/07/08/join-the-gui-generation-qtcreator/)——一个很好的使用 Qt 的工具。您将在那篇文章中看到，他们提供了一个嵌入式 Qt，可以在 Linux 平台上直接绘制到帧缓冲区。看看新产品如何在此基础上有所改进将会很有趣。当然，Raspberry Pi 及其同类也可以运行常规的 Qt。 [QtCreator 可以做的不仅仅是 Qt，](https://hackaday.com/2018/12/29/open-source-ide-for-fpgas-as-qtcreator-learns-verilog/)在一些朋友的帮助下。
