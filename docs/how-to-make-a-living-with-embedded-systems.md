# 如何以嵌入式系统谋生

> 原文：<https://hackaday.com/2019/10/12/how-to-make-a-living-with-embedded-systems/>

每个人最大的梦想之一就是做自己喜欢的事情来谋生。对于所有对嵌入式系统充满热情的黑客、制造者和 DIY 者来说，最初将嵌入式系统设计作为一种可能的职业可能是有意义的，但是如果没有这么多关于资格类型或实际获得工作机会所需步骤的信息，尝试进入该领域可能会令人生畏。

YouTuber [iAyan Pahwa]目前是一名嵌入式软件工程师，已经在该领域工作了两年，之前是嵌入式领域的微控制器、电机和编程爱好者。在下面的视频中，他[提供了他对你获得第一份工作所需要知道的事情的看法。](https://www.youtube.com/watch?v=TM8QAC-mGD8)

### **好处**

嵌入式系统的工作往往报酬丰厚，而且有很多机会从事有趣的项目。由于消费电子产品存在于所有行业，你不会缺乏任何公司寻求雇用具有嵌入式系统知识的人。

### **嵌入式硬件**

这些角色通常负责原理图布局、嵌入式电路板设计、电力电子、PCB 设计以及依赖模拟领域的任务。你需要能够使用一些 CAD 和 EDA 软件，并且有模拟调试的经验。

虽然在嵌入式硬件领域有大量的机会，但[Pahwa]主要是谈论如何打入嵌入式软件领域。

### 嵌入式软件

大多数软件工程师用 C 语言工作，从事引导加载程序、中间件或内核开发。要找的课程包括 CS50(basic C)、学习设备驱动程序编程和系统编程。

此外，学习编程微控制器是必不可少的。即使你最终进入了超大规模集成电路或电子控制自动化领域，至少学习 Arduino 编程对于深入挖掘嵌入式系统的硬件方面是非常有帮助的一步。虽然 Arduino 是用基于 C++的语言编程的，但熟悉 C 的程序员应该不会对语法有任何问题。

学习用 C 或汇编语言编写微控制器程序对于优化你的代码是必要的。能够理解计算机架构和寄存器可以增加您对微控制器内部设计以及如何在生产中使用它们的了解。你会遇到的常见问题包括堆栈指针、左右移位、移动寄存器以及如何控制内存。

除了了解工业中使用的不同处理器，了解操作系统也是有帮助的，尤其是了解 Linux 的来龙去脉。在面试中，您可能会被问到系统调用、文件系统和命令行，以测试您在 Linux 环境中的舒适度。像 Verilog 或 VHDL 这样的硬件描述符语言(HDL)也是嵌入式软件工程师需要的常见技能。

### 外卖食品

[Pahwa]更详细地介绍了学习嵌入式系统的学生希望在嵌入式系统职位面试中胜出的具体技巧，以及任何不确定他们是否真正热爱这个行业的人。

嵌入式系统工程师掌握的许多技能都是后天习得的，如果你已经是一名业余爱好者，你可能已经熟悉了嵌入式系统所需的许多工具和过程。调试汇编和使用不熟悉的微控制器的数据手册等技能只能通过实践和时间来获得，因此花时间从事项目工作并熟悉现有环境对于在该领域取得成功至关重要。

最重要的是，保持动力并寻求该领域其他人的专业知识是一笔巨大的财富。由于这个领域如此广阔，对于一个有抱负的嵌入式系统工程师来说，你会发现无限的机会和资源。
