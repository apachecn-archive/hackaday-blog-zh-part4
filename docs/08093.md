# 在我们计算的地方，我们不需要七段！

> 原文：<https://hackaday.com/2020/10/18/where-were-calculating-we-dont-need-no-seven-segments/>

自从第一个白炽灯泡或氖管在原始计算机的命令下开始闪烁以来的几十年里，人们对电子数字显示技术进行了许多尝试，但最小的公分母仍然是不起眼的七段。这可能会结束，但对[肯·雅普来说，他从 20 世纪 60 年代的夏普计算器[中获得灵感，重新创造了一个只有六段的数字显示器](https://hackaday.io/project/175396-6-segments-suffice)。

这个看似不可能的壮举是通过将六个弯曲的线段排列成数字 8 来实现的，这可以在某种程度上呈现所有的数字，但这很快就揭示了为什么额外的线段会出现。由曲线组成的数字看起来足够好，但 1、4 和 7 中的直线被对角线破坏了，零在一半的高度上显得奇怪地小。你*可以*阅读数字，但需要习惯。

降低 20 世纪 60 年代电子产品复杂性的意义在 2020 年只是一个迷人的好奇心，但我们可能不会经常看到这些显示器出现。你可以在休息时间下面的视频中看看它，如果你对激发它的夏普计算器感到好奇，那么你可以看看复古计算器博物馆中的[页面。](http://www.vintagecalculators.com/html/sharp_qt-8d.html)
