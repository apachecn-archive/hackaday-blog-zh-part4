# 最小可行四元构建展示了从零开始可以实现什么

> 原文:[https://hacka day . com/2021/08/12/minimum-viable-quad-build-shows-what-starting-from-nothing-can-complete/](https://hackaday.com/2021/08/12/minimum-viable-quad-build-shows-what-starting-from-nothing-can-accomplish/)

虽然有经验并拥有解决问题所需的大量专业知识是件好事，但从无知的立场出发去做事情也是一种解放。从零开始可以引导你走上一条人迹罕至的道路，并揭示出原本可能不会出现的解决方案。如果[【Robin Debreuil】对“最小可行四轴飞行器”](https://youtu.be/Yt-7wDAzyEA)的探索是任何例子的话，也有一些非常有趣的故障模式。

当然，最小可行产品的概念并不新鲜，它是精益方法中的核心概念，也是许多不同行业中的常见做法。创建 MVP 的想法是让一些东西工作起来，并交到用户手中，然后用户会给你反馈，告诉你所有的错误，如果你幸运的话，还会告诉你你做对了什么。该反馈通知下一个设计，这导致更多的反馈和整个迭代过程，应该设计出完美的小部件。

在[Robin]的例子中，他想造一架四轴飞行器，但不知道从哪里开始。所以他的第一个版本尽可能简单:一个带螺旋桨的马达和一个小的脂肪电池。没有底盘，没有控制电子设备——什么都没有。它的效果和预期的一样好。但是解决这个问题导致了不同的设计，其过程非常有趣——我们特别喜欢由水银倾斜开关控制的反向电机来感应姿态变化的 quad。

最终，[罗宾]采取了更传统的策略，使用微控制器和 BetaFlight 让他的冰棒棒和热熔胶无人机升空。但是，从最小可行设计开始并从那里迭代的决定是一次强有力的学习经历，与[罗宾]的另类和低调的观点一致，我们之前已经看到过[他使用铋来脱焊](https://hackaday.com/2021/06/02/make-your-desoldering-easier-by-minding-your-own-bismuth/)和[他的刮掉 PCB](https://hackaday.com/2020/07/10/making-pcbs-the-easy-way/)。
