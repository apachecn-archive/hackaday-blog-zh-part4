# 不是在网上

> 原文:[https://hackaday.com/2022/07/02/not-on-the-internet/](https://hackaday.com/2022/07/02/not-on-the-internet/)

每当你需要知道什么的时候，你就在网上查找，对吗？使用你选择的搜索引擎，你输入几个关键词，按回车键，你就搞定了。任何数据表、任何协议规范、任何模糊的运行时错误、任何时间。见鬼，你经常可以找到一些示例代码来实现你想要的东西。大约一分钟后。

找到所有技术性的东西实在是太容易了，以至于我认为这完全是理所当然的。事实上，我完全忘记了我们生活在一个黑客的乌托邦里，直到几天前的一个晚上，当它再次发生:我想找到一些不在互联网上的东西。现在，公平地说，它可能就在那里，我只需要挖掘得更深一点，但是*而不是*立即找到随机深奥问题的答案的震惊提醒我，当我们立即找到答案时，我们实际上有 99.99%的时间是多么幸运。

干得好，超级书呆子的全球蜂巢思维！这是互联网的创始梦想之一，所有的信息对任何地方的任何人都是可用的，这基本上是可行的。不要介意我们可以播放电影或与地球另一端的人进行电信联系——当我想要一个 Python 库来解码堪萨斯城标准音频数据时，它就在我的指尖。详细的 SCSI 规格？检查。

但你会问，我的搜索是什么？Kristina 和我谈到了 Teddy Ruxpin，我认为磁带上伺服轨道的规格肯定已经被逆向工程和很好地记录了下来。我仍然确信它是——我只是震惊于我不能立即找到它。上次我遇到这种情况时，是构成 Speak & Spell 的芯片的数据表，结果是我只需要更加努力地挖掘。所以我还没有放弃希望。

内心深处，我有点高兴我在互联网上找到了一个漏洞。这给了我和克里斯蒂娜一个借口，让我们自己对格式进行逆向工程。有时候无知是福。但在剩下的时间里，当我真的想要一个小众技术问题的答案时，谢谢大家！
