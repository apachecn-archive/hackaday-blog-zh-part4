# 锤子找钉子

> 原文:[https://hackaday.com/2020/08/15/hammer-seeks-nail/](https://hackaday.com/2020/08/15/hammer-seeks-nail/)

人们有时会说“当你有一把锤子时，所有的东西看起来都像钉子”，好像这是一件坏事。在维基百科上，他们称之为 [*乐器定律*](https://en.wikipedia.org/wiki/Law_of_the_instrument) 或*马斯洛的锤子*，并称之为认知偏差。但是我喜欢锤子…

我正在研究一个新工具，一个四轴热丝泡沫切割机[大致基于这个设计](https://www.rcgroups.com/forums/showthread.php?2915801-4-Axis-Hot-Wire-CNC-%28Arduino-Ramps1-4%29-Complete-Solution)，但是目前为止是在我的地下室里用材料建造的。我希望它主要是为遥控飞机制造机翼，这样我就可以研究机翼和建造方法等等。但是在建造这个新“锤子”的中途，我已经有了可以用它建造其他项目的有趣想法。典型的寻钉行为。

[![](../Images/d529c59d34ec5a7f77ab7233760d66fb.png)](https://hackaday.com/wp-content/uploads/2020/08/32803608_8839a75be5_o.jpg) 这些想法让我重新考虑我的锤子的设计。我最初想把它建得低一些，因为我不太可能想要切割高于 50 毫米左右的机翼部分。但是，当我想到切割巨大的字母来装饰我儿子的房间，或者可能是船体的一部分时，这意味着一个明显更高的切割机，随之而来的是复杂性。

所以在这里，我同时遭受了马斯洛的锤子和范围蠕变，但我并不为这些“疾病”感到难过。玩几个数控热线切割机的手动原型扩大了我的设计词汇量；我想到了几个很酷的项目，我以前根本没有想到过。拥有工具扩展了你可能的构建方式，不管有没有认知偏见。

一个人的范围蔓延是另一个人的“充分实现项目的潜力”。我很确定无论如何我都会造出这台机器的第二个版本，所以如果第一稿有高度限制可能没什么大不了的，但是思考高度问题的过程实际上让我有了一个更好的设计，即使是对短切刀来说。(张力由外部弓提供，而不是由垂直数控塔承受。我完成后会把这个项目写下来。但这不是重点。)

也许我们不应该哀叹马斯洛的认知偏差，而是应该庆祝同一枚硬币的另一面:钉子非常有用，拥有一把锤子的简单事实可以让你充分欣赏它们，反过来拓展你的能力。至于范围蠕变？只要我在下周的假期里完成这个项目，一切都很好，对吗？
