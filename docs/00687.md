# Linux，没有 Git 因素

> 原文：<https://hackaday.com/2018/09/17/linux-without-the-git-factor/>

Linux 开始于 20 世纪 90 年代，是 Linus Torvalds 的学生项目。几十年来，他在指导自己的创作的同时获得了名人的地位，但与此同时，他也在 Linux 和软件社区中赢得了不同的声誉。他的爆发和咆哮是出了名的，其中一些变得相当个人化，因此不难发现内核开发人员或潜在的内核开发人员已经拒绝了这个项目。

然后阅读[Linus] 的更新，作为他定期交流的一部分，确实令人耳目一新，他在其中承认他有问题，并表示他正在花时间寻求帮助。有一个[对内核维护行为准则](https://git.kernel.org/pub/scm/linux/kernel/git/torvalds/linux.git/commit/?id=8a104f8b5867c682d994ffa7a74093c54469c11f)的伴随更新，这表明这很可能标志着那个环境的巨大变化，我们也希望挽救【莱纳斯】在这方面的声誉。

> “我在电子邮件中轻率的攻击既不专业也不必要。尤其是当我把它变成私人恩怨的时候。在我寻找一个更好的补丁时，这对我来说是有意义的。我现在知道这是不对的，我真的很抱歉。”

Hackaday 社区的读者中，Linux 用户的比例远远高于平均水平。即使是那些使用带有 BSD、Windows 或其他内核的桌面操作系统的读者，也几乎肯定会在某个地方安装 Linux 内核，无论是在他们的手机、机顶盒、孩子的玩具，甚至是他们的家用电器中。当然，很大一部分互联网运行在 Linux 上。我们继续吸引和留住优秀的人才，为开发和维护 Linux 内核做出贡献，这符合我们所有人的最佳利益。

我不想把[Linus]捧得比其他许多为 Linux 及其成功做出贡献的人更高，他对我们社区的贡献是无法估量的，看到他的另一面让人很不舒服。这是朝着正确方向迈出的一步，为个人攻击和行为道歉，并寻求改变这种行为。我们敦促其他人以他为榜样，我们确信每个群体有时都有自己的个性问题，进行一些修复永远不会太晚。

当 Linus 离开去进行自我改进时，资深内核开发人员 Greg Kroah-Hartman 将接管内核维护人员的工作。
