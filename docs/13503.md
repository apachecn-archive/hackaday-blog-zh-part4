# Arduino 和 Git:两种视图

> 原文:[https://hackaday.com/2022/04/29/arduino-and-git-two-views/](https://hackaday.com/2022/04/29/arduino-and-git-two-views/)

如果不接触 Git(版本控制管理系统),你就无法进行太多的开发。一部分原因是因为大量代码存在于使用 Git 的 GitHub 上，尽管如果你只想下载代码，你不需要了解这些。如果你需要尝试一下，Torq 博士有一本很好的关于使用 Git 和 Arduino IDE 的入门书。

你可能认为如果你自己开发，你不需要 Git 这样的东西。然而，使用版本控制系统是非常方便的，特别是如果你正确使用的话。田野里有只虫子？固件的版本是什么？您可以使用 Git 立即获得该时间点的源代码副本。一个功能坏了？很容易看出到底发生了什么变化。因此，即使你不在团队中工作，控制源代码也是有好处的。

如果您已经在使用更高级的 IDE，Git 很可能已经集成到您的环境中，或者至少有可能。如果您对命令行过敏，也有很多 GUI 工具可以使用 Git。Git 的一个好处是您的本地存储库只是一个目录。你不需要竖起一个特殊的服务器或任何东西来使用它。

我们不认为这是直接相关的，但[Andreas]最近有一个关于 Git 和 Arduino 的视频。它不是那么详细，但它确实有一些好东西，值得花时间观看。你可以在下面看到。

在过去，我们已经用 Git 做了一些奇怪的事情。注意 Git 是版本控制系统，而 GitHub 是网站。然而，如果你是一个铁杆命令行用户，你也可以从那里[操纵 GitHub](https://hackaday.com/2020/02/15/github-goes-gui-less/) 。
