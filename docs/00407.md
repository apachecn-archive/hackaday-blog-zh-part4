# 如果你计划建造自己的航天飞机…

> 原文：<https://hackaday.com/2018/08/21/if-you-are-planning-on-building-your-own-space-shuttle/>

有史以来最复杂的机器之一是美国航天飞机(技术上，STS 或太空运输系统)。尽管标题如此，我们怀疑是否有人会复制它。然而，关于航天飞机的航空电子设备——操作机器的电子设备——最有趣的事情之一是，作为一个政府项目，关于它如何工作的可用材料数量惊人。美国宇航局有一个网页收集了对飞行器航空电子设备的描述。如果你对实际的火箭科学更感兴趣，只需[倒退几级](https://spaceflight.nasa.gov/shuttle/reference/shutref/index.html)。

不过，我们要提醒你，如果你从未从事过太空硬件，一些设计选择会显得很奇怪。这有两个原因。第一，环境很陌生。你必须处理高加速度、冲击、振动和辐射，以及其他事情。另一个原因是，由于测试和简单的繁文缛节，设计和部署之间的时间太长，几乎可以肯定，您将使用几乎过时的技术进行部署。

一个很好的例子是轨道飞行器的 GPC 或通用计算机。他们有五个人——在做任何事情之前，至少要有三个人同意。GPC 的早期版本使用磁芯存储器。这项技术在设计时就已经过时了，但在设计阶段，它是确保内存不会受到辐射影响的最佳方式。1991 年，一次重大升级确实用基于半导体的存储器取代了内核。

美国宇航局网站上的信息有点高级，但仍然很详细。如果你想就航天飞机的航空电子设备进行一些真正的核心讨论，KLabs 网站在转移到私人网络服务器保持运营之前失去了美国宇航局的资助。如果你曾经想像航天飞机程序员那样尝试用 HAL/S 编程，你可以在那里得到文档。下面还有一本[小册子，里面有很多信息](https://ntrs.nasa.gov/archive/nasa/casi.ntrs.nasa.gov/19900015844.pdf)，你可以从 NASA 下载，还有一段你可能会喜欢的视频概述。

虽然我们怀疑是否会有人试图建造自己的航天飞机，但这仍然是一笔巨大的信息财富。当然，[俄罗斯人曾经尝试过](https://hackaday.com/2016/04/08/stolen-tech-the-soviet-shuttle/)，并且可以说创造了一个更好的系统。如果你真的做了一个副本，已经有来自海因斯的[车间手册](https://smile.amazon.com/NASA-Space-Shuttle-Manual-Construction/dp/1844258661)。
