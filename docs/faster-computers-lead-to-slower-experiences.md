# 更快的电脑导致更慢的体验？

> 原文：<https://hackaday.com/2019/03/22/faster-computers-lead-to-slower-experiences/>

有没有那种奇怪的感觉，事情和以前不太一样了？不是说一台新洗衣机的塑料部件比一台 40 年前的洗衣机多。更像是“我的笔记本电脑可以运行数百千兆次浮点运算，但当我滚动时，感觉并不好。”这种平滑度的感觉可能基于几个因素，包括系统延迟。几年前[danluu]也有这种感觉，并且[测量了“我在过去几个月遇到的设备”的延迟](http://danluu.com/input-lag/)(基于这个列表，他过着比我们更有趣的生活)。事实证明他的预感客观上是正确的。他的文章精彩地深入探讨了各种设备的工作方式和原因，以及硬件和软件对延迟的影响。

让我们明确一下“延迟”在这个上下文中是什么意思。[danluu]正在检查用户输入和屏幕上的一些响应之间的时间。对于桌面系统，他测量了击键，对于移动设备，他测量了浏览器的滚动。如果你是在 Hackaday(或者可能是在[老式电脑节](https://hackaday.com/2019/03/19/this-weekend-vintage-computer-festival-pacific-northwest/))上，图表顶端明显矛盾的原因可能是显而易见的。

问:为什么一些旧系统比几十年后生产的设备速度更快？答:旧系统没做多少事！复杂的多任务操作系统不再同时处理数百件事情，CPU 的全部注意力都集中在正在运行的用户进程上。这里有明显的实际缺点，但它确实减少了上下文切换！

在某种意义上，[danluu]描述的这种复杂性是我们如何用编程解决问题的核心。写代码都是关于抽象的。虽然任何程序都可以直接用机器码编写，并根据个人机器的硬件配置进行定制，但这对开发人员和用户来说都很不方便。因此，随着时间的推移，在用高级编程语言编写的更好的界面后面，添加了一层又一层的糖来隐藏原始硬件。

有一个内核来处理最低层，然后是添加热插拔系统、电源管理、可插拔模块和驱动程序基础架构等的层，而不是针对确切的硬件配置编写每个程序。当考虑编程问题的解决方案时，方法总是递归的:您可以解决问题，或者添加一个抽象层并重新构建它。后者的足够层数使前者变得微不足道。但这完全是抽象的。

[danluu]的观察是，我们现在才开始掉头，再次达到低延迟，但这一次是靠蛮力！现代延迟解决方案很大程度上看起来像是越来越奇特的显示技术和复杂的优化，从 UI 绘制功能一直延伸到芯片，而不是删除软件和系统基础设施。事实证明，软件复杂性在用户体验和易于开发方面的好处在大多数时候是值得的。
