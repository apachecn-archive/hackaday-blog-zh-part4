# 使用 Calcutron-33 的 10 种程序员

> 原文：<https://hackaday.com/2022/12/07/the-10-kinds-of-programmers-that-use-calcutron-33/>

有趣的是，如果你观察足够长的时间，事情往往是周期性的。过去，有些计算机不用二进制，而是用十进制。这对像 TUTAC 或 CARDIAC 这样的教育计算机来说尤其如此，但也有真正的十进制硬件。然后每个人都认为二进制更有意义，现在很难找到一台不使用它的计算机。

但是[Erik]已经为 [Calcutron-33](https://github.com/ordovician/calcutron) (一种“十进制 RISC”CPU)编写了模拟器、汇编程序和调试器。为什么？这个想法是提供一个教学平台来解释汇编语言的概念给那些可能偶然发现二进制数的人。一旦他们理解了 Calcutron，他们就可以带着一定的信心转向更传统的 CPU。

为此，有几篇文章涵盖了[基本架构](https://erikexplores.substack.com/p/the-calcutron-33-decimal-based-computer)、[指令集、](https://erikexplores.substack.com/p/the-calcutron-33-instruction-set)以及[如何为机器编写汇编](https://erikexplores.substack.com/p/writing-calcutron-33-assembly-code)。除了全部使用十进制以外，CPU 与现代微处理器有许多共同之处。

已经有几个版本的虚拟机进行了各种改进和错误修复。老实说:我们钦佩这项工作及其范围。但是，如果您已经了解二进制，这可能不是您的最佳选择。更重要的是，也许你应该在处理汇编语言编程之前理解二进制，至少在现代是这样。尽管如此，它确实涵盖了许多适用的领域。

当计算机时间过于昂贵而不能浪费在学生身上时，像 TUTAC T1 和 T2 T3 这样的自制计算机风靡一时。还有来自计算机传奇人物[唐纳德·克努特](https://hackaday.com/2016/06/06/learn-to-program-with-literate-programming/ddddd)的混音。
