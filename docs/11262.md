# 过度设计智能门铃

> 原文:[https://hack aday . com/2021/09/12/over engineering-a-smart-门铃/](https://hackaday.com/2021/09/12/overengineering-a-smart-doorbell/)

刚从中世纪辉煌的比利时城市根特回来，我们将从 Hackerspace Gent 组织的 Newline 黑客大会上为您带来更多内容。[Victor Sonck]在他房子的顶层工作，因此需要一个门铃通知器。他的解决方案出人意料，正如他承认的那样，过度设计了，[使用来自麦克风的音频流的机器学习来检测门铃的声音](https://www.youtube.com/watch?v=GMLE4b0AKFE)。

在确定用门铃向亚马逊出售自己的灵魂并不是一个合适的解决方案后，他接下来查看了他现有的门铃。我们中的一些人可能会直接连接到它的电源来感知按钮何时被按下，但我们有点高兴他走了过度工程化的路线，因为这意味着我们被告知机器学习如何工作以及它如何应用于音频。最终的结果有时可能是由勺子碰到麦片盘子引发的，但是因为他能够证明它的工作，我们认为这可以被称为成功。如果你想深入了解，你可以在他的 GitHub 库中找到更多的[。](https://github.com/thepycoder/doorbell_detector)

你会如何过度设计一个门铃？使用 GNU 无线电和过滤器？或者是一个有绳子和滑轮的 Rube Goldberg 机器？一如既往，评论是开放的。
