# 使用 GitHub 动作煮咖啡

> 原文：<https://hackaday.com/2022/12/10/using-github-actions-to-brew-coffee/>

越来越难想象一款现代高端设备不具备某种程度的互联网连接。如今，似乎除了最便宜的冰箱、空气净化器和微波炉之外，所有东西都包含无线功能——不幸的是，它们往往没有得到很好的实施，或者在专有系统的背后。[Matt]最近购买了一台具有蓝牙功能的高端咖啡机，结果证明它几乎没有用，他开始对他的咖啡机进行逆向工程，并通过 GitHub 发送命令使其适应工作。

由于这款咖啡机的无线连接和应用程序如此糟糕和不可靠，[马特]首先需要深入了解蓝牙低能耗(BTLE)技术。在嗅出流量并识别出咖啡机后，他开始在 Rust 中为它建立一个界面。一旦他能够向它发送命令，下一步就是将其与 GitHub 集成，这样在 GitHub 界面上提交问题就可以通过蓝牙将命令从附近的计算机发送到咖啡机，比咖啡机最初提供的更可靠。

使用[Matt]的方法，任何人都可以使用其中一台 Delonghi Dinamica Plus 咖啡机，重新激活其无线功能。虽然我们希望销售这种优质产品的人会花很少的时间，并确保额外的功能实际上是可行的，但这种低门槛似乎是公司难以超越的普遍现象。但不需要拿起这样一台昂贵的机器就能远程冲泡一杯咖啡。你可以用一台非豪华的咖啡机和一些基本的无线硬件很容易地做到这一点。
