# 平衡设计以及如何知道何时停止优化

> 原文:[https://hack aday . com/2021/03/13/balanced-design-and-how-know-when-to-quit-optimizing/](https://hackaday.com/2021/03/13/balanced-design-and-how-to-know-when-to-quit-optimizing/)

我买了一台相对便宜的 6040 数控机床，花了大部分周末时间让它工作，然后切割东西，学习工具链，并进行后续改进。大概我 90%的机器时间都在做改进。这并不是说这台机器不好，我得到的是带有滚珠丝杠和相当坚固的框架的版本，而是它不知何故不能作为一个整体一起工作。这只是一个不可思议的不平衡设计。

让我们从主轴电机开始。这是一个 2.2 千瓦的水冷怪兽，能够将数吨的工作放入一个工件中，并以非常高的速度旋转。然而，为了跟上高速主轴，移动它的电机也必须能够高速运转——如果你不是数控极客，这是一个[进给和速度](https://en.wikipedia.org/wiki/Speeds_and_feeds)的事情。他们不能。相反，套件中的步进电机是为低速时的最大力量而设计的。这对于一些机器来说是有意义的，但是对于像这样有一个稍微灵活的 X 轴的机器来说，这也是浪费。框架只是不能处理电机能够处理的低端咕噜声，所以它也不能利用主轴的动力。设计无处不在。

在过去两个月的周末，我一直在反复问自己“现在我的限制因素是什么？”致力于修复它，运行它，然后再次问这个问题。这是一个很好的通用程序，我相信它能让我以最少的时间、金钱和精力获得我想要的机器。

起初，它是带有模拟 USB 并行端口的驱动程序硬件/软件，所以我用运行 GRBL 的 Arduino 替换了控制器，直接焊接到后面的 DB-25 上。至少它能以足够快的速度发出脉冲来命令马达运转，但它们在高速运转时仍然会熄火。把步进电机换成一对高速电机只花了我 40 欧元，这让你想知道他们为什么不一开始就装上正确的电机。现在机器的速度足以利用高速主轴，我飞过胶合板和塑料，没有留下烧伤的痕迹。这是一场不花多少钱的巨大胜利。

最后一个前沿领域是从铝中获取大量份额。主轴可以做到这一点，但我担心我会遇到框架在 X 轴上的刚性。不管出于什么原因，他们在 X 上使用了无支撑的杆，这比由更多金属支撑的轴要灵活得多。这就是限制因素可能实际上是我的时间和耐心，而不是钱。我实在不忍心把这东西再拆再装。所以现在，它将是小啃，利用机器的速度，如果还没有主轴的全部马力。

不过很奇怪，因为这台机器*是*一捆好零件。只是他们没有被选择以最佳方式合作；框架不与步进电机一起工作，步进电机不与主轴一起工作。如果他们按照我的程序问“限制因素是什么？”他们本来可以节省自己的€100 只运输它与一个懦弱的主轴，这将是一个平衡，如果贫血，机器。或者他们可以用合适的马达来提高速度。或支持更多咕噜声的轨道。或者两者都有！

我永远也不会知道他们为什么放弃优化他们的设计。也许他们从来没有通过缓慢的 USB/并行端口速度？但我已经接近人生的终点，我能说出来是因为限制因素不再是简单的升级，甚至不再是单纯的金钱，而是我自己的意志力。

当你在浓雾中的山顶时，你怎么知道呢？你朝任何方向迈出的一步都会让你走下坡路。如何判断您对项目的状态是否满意？当你没有必要或欲望去进行下一个最明显的改进时。
