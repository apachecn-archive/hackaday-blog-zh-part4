# 另一个魔方机器人虽然简单，但速度很慢

> 原文：<https://hackaday.com/2022/06/13/another-rubiks-cube-robot-is-simple-but-slow/>

[AndreaFavero]说, [CuboTino](https://www.instructables.com/CUBOTino-Autonomous-Small-3D-Printed-Rubiks-Cube-R/) 强调简单和节约成本，而不是速度。然而，在大约 90 秒内解决难题仍然比我们能做的要好。勇敢解算器使用 Pi 和相机来了解立方体的外观，然后通过解算器运行它来确定如何移动。

看了下面的视频，我们对这些机制印象深刻。倾斜的表面解决了一些问题，使操作更容易。按照机械装置的排列方式，只需要一对伺服系统就可以随心所欲地翻转立方体。

显然，在解出魔方所需的 90 秒中，大约有 30 秒是在看它。这不是一个快速的下午 3D 打印。有 13 个独立的部分，尽管所有部分都应该很容易打印。

当然，我们以前见过很多立方体解算器。但是看到不同的人如何处理同一个问题总是很有趣的。顺便说一下，这台机器是我们在之前[从【AndreaFavero】看到的早期机器的进化。如果你认为这些说明看起来有点简单，请注意，你必须下载 PDF 格式的完整说明。所有的细节都在那里。](https://hackaday.com/2022/02/02/forget-sudoku-build-yourself-a-minimalist-rubiks-solver-robot/)

如果你想构建更复杂的东西，[你可以走得更快](https://hackaday.com/2016/01/27/robot-solves-rubiks-cube-in-just-one-second/)。正如您将看到的，要获得额外的速度需要做更多的工作。
