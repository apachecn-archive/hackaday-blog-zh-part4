# 简单、可扩展、开放

> 原文:[https://hackaday.com/2022/04/16/easy-extensible-open/](https://hackaday.com/2022/04/16/easy-extensible-open/)

我是一个超级 DIY 者。能自己造的东西我不喜欢买。但老实说，从完成任务的角度来看，这并不总是能优化我的时间分配。有时候，如果你心中有一个更大的项目，正确的方法是快速的方法，而快速的方法是购买一些已经可以工作的东西。但是，当某样东西本身不可破解时，你最好确定它能做你需要的，以及你可以合理预期将来需要的，开箱即用。这就是可扩展性的用武之地。

很难找到既易于新手使用，又可扩展给高级用户的产品。首先，单独勾选这些框中的任何一个都很困难，所以两个都要加倍努力。但是我的另一个潜在的怀疑是，设计师倾向于在头脑中有一个最终用户，也许只有一个最终用户，这就是问题所在。为新手设计时，方便是王道。或者，如果以专业人士为目标，您可以最大限度地提高灵活性，但可能会以牺牲固有的复杂性为代价。

如果你愿意，有一条出路，一个欺骗代码。这使得这个项目开源。如果你愿意，继续对新用户隐藏复杂性——只要专业人员能够钻研原理图或源代码，她就会自己想出如何扩展它。开放性解放了设计者，让他们在不牺牲灵活性的情况下，更容易使用。

我认为这种简单和可扩展的结合，通过开放性，是 Arduino 成功的根本驱动力。在表面层上，有一些只是做你想做的事情的库和带有访问它们的例子的下拉菜单。但是当你需要直接使用芯片的硬件外围设备时，没有什么能阻止你。对于整个社区来说，所有代码都是公开可用的这一事实意味着扩展基础很容易——我们不要拐弯抹角了，社区的库、教程和示例项目是该平台成功的真正原因。

看看你的周围，当你做下一个非 DIY 的捷径购买时，要小心。好用吗？你能让它做它还没做的事情吗？仅仅是两个简单的要求，但是如果你两者都想要的话，它们似乎可以淘汰很多产品。然后看看那些既简单又灵活的——它们也是开放的吗？至少在我的小世界里，答案几乎总是“是”。
