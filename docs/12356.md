# 批评自己的 PCB 设计的好处

> 原文:[https://hack aday . com/2022/01/03/the-benefits-of-criticing-your-own-PCB-designs/](https://hackaday.com/2022/01/03/the-benefits-of-critiquing-your-own-pcb-designs/)

在最近的一个回顾视频中，来自 Phil 实验室的[Phil]回顾了他早期的一些 PCB 设计，并对这些设计中他喜欢和不喜欢的地方进行了评论。尽管才过了几年，他还是发现了很多问题。从原理图中不良和不一致的格式，到草率和完全损坏的 PCB 布局。这是对多年经验教训的精彩回顾。

[Phil]对原理图中各部分和页面的清晰标签和组织的重要性进行了评论，以使模块的功能显而易见。其它经验包括:标记网络以简化 PCB 布线；充分利用 PCB 平面；获取原理图中所有元件和布局的相关信息作为注释；将去耦电容连接到相关引脚。

虽然我们倾向于忘记旧的项目，但偶尔看看它们，看看(希望)我们这些年的进展，会非常有趣。在[Phil]的案例中，看到从带有 THT 元件的基本双面板到带有 STM32 MCUs 的多层板的转变是非常有趣的。
