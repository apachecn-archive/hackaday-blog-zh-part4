# Blinky 项目是 6502 一路向下

> 原文：<https://hackaday.com/2022/12/24/blinky-project-is-6502s-all-the-way-down/>

事实上，您可能会发现自己编程的任何平台都有一些运行延迟的简单方法。[乔伊·谢泼德]在最近的一个项目中变得相当有创意，尽管依赖于一个相当愚蠢的嵌套方法，我们称之为 [6502s。](https://joldosh.blogspot.com/2022/12/pic326502-blinky-robot.html)

这个项目是一个简单的 PCB，形状像一个机器人，有闪烁的 LED 眼睛。通常情况下，你可以简单地使用通常的 sleep()或 delay()功能来控制眨眼频率，但[Joey]在这一点上偏离了轨道。相反，板上的 PIC32 运行用 MIPS 汇编编写的 6502 仿真器。这个仿真的 6502 然后负责运行在 6502 汇编中编码的另一个 6502 仿真器，等等，直到有 6502 仿真器在不起眼的微控制器上运行六个深度。最里面的模拟器运行一个简单的程序，在一个简单的循环中闪烁 LED 眼睛。然而，由于运行六个模拟器的开销，眼睛只能以大约每两秒一次的速度眨眼。

这是一种有趣而复杂的编写眨眼程序的方式，我们为[Joey]所付出的努力鼓掌。我们认为这是学习 PIC32 编程以及一般仿真的好方法。与此同时，如果你正在开发自己的模拟器壮举，[一定要让我们知道！](http://hackaday.com/submit-a-tip)
