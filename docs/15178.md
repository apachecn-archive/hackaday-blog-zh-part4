# 在传感器功耗方面，越少越好

> 原文:[https://hack aday . com/2022/10/25/少即是多当它涉及到传感器功率/](https://hackaday.com/2022/10/25/less-is-more-when-it-comes-to-sensor-power/)

过去，微控制器的成本是将大脑放入一切事物的一大阻碍，但那些日子已经一去不复返了。甚至 32 位 CPU 现在也足够便宜，你可以把它们扔进任何东西里。现在最大的因素大概是动力。你真的想每隔几周就给你的电动马桶盖充电或者换电池吗？一家名为 [Everactive](https://everactive.com) 的公司希望你使用他们的传感器平台抛弃你的电池，他们声称可以从各种来源收集能量，他们即将推出他们的第一个[开发者套件](https://go.everactive.com/edge-developers)。

该传感器可以测量三个轴上的温度、湿度、压力、磁场和加速度。该设备声称可以从无线电频率、振动、小温差或光线中获取能量，即使在室内也是如此。我们的猜测是，传感器包运行的时间非常少，当您无线轮询设备时，传入的无线电信号为通信提供电源。该公司声称其设备比竞争对手的解决方案耗电少 1000 倍。

我们不确定这些说法是否有根据，但我们确实同意电池是嵌入式系统的祸根。然而，在某些情况下——比如自动售货机——电源是可用的，在其他情况下，低功耗传感器可能不需要电池，但其他用途仍然需要电池。例如，感应水箱以打开加热器，加热器仍然需要电源。

然而，在其他情况下，拥有一个自供电的传感器包将是最好的选择。我们只是不认为它适用于所有情况。我们确实喜欢能量收集，但有时如果你了解你在建造什么，你可以做得更好。例如，有[自动冲洗阀，其利用水流产生足够的电力来操作装置](https://www.totousa.com/technologies/ecopower)。如果储能设备(如可充电电池或超级电容器)电量不足，你只需冲水充电。
