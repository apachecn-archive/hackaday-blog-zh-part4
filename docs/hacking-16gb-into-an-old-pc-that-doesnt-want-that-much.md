# 将 16GB 的内存装入一台不需要那么多内存的旧电脑

> 原文：<https://hackaday.com/2019/04/06/hacking-16gb-into-an-old-pc-that-doesnt-want-that-much/>

从标题来看，你可能会认为这篇文章是一个蹩脚的故事，讲的是有人插上了内存，也许还在更新 BIOS。这就是你错的地方。《道格·布朗下城》有一个更有趣、更有教育意义的故事。

[Doug]发现他的主板额定最大 8 GB，并决定[尽管有限制](https://www.downtowndougbrown.com/2019/04/adventures-of-putting-16-gb-of-ram-in-a-motherboard-that-doesnt-support-it/)，他还是让 16 GB 的 RAM 工作。他更新了 BIOS，并且成功了——在 Linux 中。他能够看到所有的内存，并且测试良好。如果是这样，你就不会在这里读到了。当他试图启动 Windows 10 时，这个故事变得有趣起来，它拒绝了，显示了它更友好、更温和的蓝屏死亡。对许多人来说，这将是故事的结尾，特别是因为 Windows 10 没有给你太多关于它为什么崩溃的信息。

像许多问题一样，这个问题必须像剥洋葱一样剥下来。首先要做的是更改 Windows 注册表，允许蓝屏输出一些老版本 Windows 中的技术信息。错误代码表明该问题与 BIOS 报告内存区域重叠有关。

随着在 Linux 中的一些调查，其日志文件获得了更多的 BIOS 信息，[道格]意识到 E820 接口正在返回一个与 ACPI 的信息冲突的内存区域。看起来好像主板在为 PCI 操作保留 8 GB 范围的顶部空间，这在系统的(现在更大的)内存中打了一个洞。关闭 BIOS 中的设置解决了这个问题，但只是因为它使 Linux 和 Windows 都只能看到 4GB 的内存。这也不会是一个非常有趣的故事。[Doug]从理论上说，如果他能够将映射区域移动到 16 GB 范围的顶部，事情将会成功。

接下来是对用于读取和更改系统信息的 Linux 工具的精彩阐述。他让它工作了吗？看看帖子就知道了。但我们会告诉你，他确实设法让 grub 修补了他的系统信息。
