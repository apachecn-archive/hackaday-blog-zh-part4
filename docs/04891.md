# 满足您串行访问需求的微型终端

> 原文：<https://hackaday.com/2019/11/22/a-tiny-terminal-for-your-serial-access-needs/>

一些计算设备已经消失得无影无踪，例如，调制解调器并不是一个可以留住许多黑客的设备，因为他们正在努力在 Arduino 上完美地模仿德高望重的 Hayes。但是有一些其他的设备仍然保持着魅力，甚至在它们离开职业舞台多年之后。串行终端就是一个很好的例子。一旦在许多桌子上发现它，因为它提供了一种进入小型计算机的方法，它很久以前就被终端模拟器或简单的其它进入计算服务如 web 的入口所取代。但是在我们的社区中仍然可以找到大量的串行终端，例如[RedHog]的小[在 ESP8266 平台](https://innovationgarage.github.io/TTyGO/)上模拟 VT220。

如果你希望一个全绿色的屏幕和键盘插入一个巨大的 D 连接器串行端口，那么你会有点失望，因为一个小小的屏幕和开发板上的几个按钮或 Arduboy 不允许任何 1337 vi 的掌握，即使它的屏幕键盘。看看吧，我们在休息时间下面放了一段视频。但尽管如此，它显示了前景，具有使用逻辑级串行端口或 USB 串行接口的能力。我们可以看到，它可能会被用作串行显示，但如果它以某种形式使用 8266 的机载 WiFi，我们会很感兴趣。如果你想尝试，你可以用 GitHub 上的[代码来做。](https://github.com/innovationgarage/TTyGO)

这并不是我们之前在这里看到的第一个这样的终端，当然，其中一些已经[非常实用](https://hackaday.com/2017/04/06/hackaday-prize-entry-pocket-serial-terminal/)，而另一些我们只是想拥有[，因为它们*酷*](https://hackaday.com/2010/02/24/oscilloscope-doubles-as-a-serial-terminal/) 。
