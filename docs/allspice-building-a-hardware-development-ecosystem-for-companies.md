# 多香果为公司构建硬件开发生态系统

> 原文：<https://hackaday.com/2022/02/27/allspice-building-a-hardware-development-ecosystem-for-companies/>

在我们的“硬件开发变得严肃”新闻中，我们[最近了解了 AllSpice](https://techcrunch.com/2022/01/31/allspice-thinks-hardware-developers-lack-their-own-github-so-it-is-building-one/) ，一家为公司构建硬件开发协作基础设施的初创公司。硬件开发人员擅长为自己构建硬件工具，但在软件方面可能并不总是如此，AllSpice 的目标是在“硬件公司”层面填补这一空白。如今，经常发生的是软件开发工具和集成被重新用于硬件需求，结果并不总是像在软件世界中那样出色。换句话说，AllSpice 正在从软件行业的积极成果中学习，并建立一个平台，从这些工具中获取最佳部分，旨在在目前缺乏硬件团队经验的领域中获得类似的积极成果。

多香果正在构建的似乎是一个伞状平台，旨在增强、集成和挂钩一系列不同的已开发平台，如 GitHub、GitLab、吉拉(以及其他一些平台)，并添加大规模硬件开发商无力自行维护和开发的急需功能。“通过截图进行设计审查”在硬件界并不是闻所未闻，可能我们每个有硬件协作经验的人都参与过。在公司范围内，有无数与硬件相关的问题需要解决和改进。

当试图将您的项目扩展为一个产品时，您可能最终会发现自己在一个团队中，所有人都在从事同一个硬件项目。在这样的合作中，有一个简单的原则——你需要有工具来改善所有相关人员之间的沟通，而拥有一个硬件定制的平台对此至关重要，尤其是当你的产品增长到一系列不同的版本和 SKU 时。正如 GitHub 集成了编译和测试代码的功能，当有人向您发送拉请求时，我们用于协作硬件开发的工具应该以开发人员友好的方式跟踪我们的 BOM、PCB 和原理图的变化。如果这个项目不能成长为一个我们这些普通黑客可以使用的平台，我们应该希望它至少有足够的影响力来积极地影响我们目前用于硬件协作需求的工具。

我们时不时会涉及到合作开发的产品，甚至在某个时候还有一个[“在 Hackaday 上开发的”系列](https://hackaday.com/tag/developed-on-hackaday/)！协作设计项目一个非常流行的例子是会议徽章，每一个都是许多人劳动的典型成果。谈到“成长为一个产品”方面，数量出乎意料的美国黑客以此为生，要么是通过在 Tindie 上销售的[，要么是通过用我们的产品做大](https://hackaday.com/2019/03/04/from-software-to-tindie-hack-chat-with-brian-lough/)的[。](https://hackaday.com/2021/11/02/hardware-project-becomes-successful-product-for-solo-developer/)
