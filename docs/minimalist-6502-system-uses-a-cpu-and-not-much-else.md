# 极简的 6502 系统使用了一个 CPU，没有太多其他的东西

> 原文：<https://hackaday.com/2022/11/22/minimalist-6502-system-uses-a-cpu-and-not-much-else/>

中央处理器，或称 CPU，是任何计算机系统的心脏。但这肯定不是唯一的部分:你还需要 RAM、ROM 和至少一些外围设备，才能把它变成一个完整的系统，实际上可以做一些有用的事情。现代微控制器通常将这些功能的一部分或全部集成到一个芯片上，但传统的 CPU 没有:它们本来应该与几十个其他芯片一起放在主板上。这就是为什么[c0pperdragon]的最新项目，[single bread board computer 是如此惊人的设计:](https://github.com/c0pperdragon/SingleBreadboardComputer)辅助其 6502 CPU 的只是四个配套芯片。

整个系统只占用一条无焊试验板。在 CPU 旁边，我们发现 32 KB 的 SRAM，32 KB 的 flash 和一个时钟振荡器。第五个芯片是一个 74HC00 四路双输入与非门，它被用作一个非常小的粘合逻辑，将所有东西连接在一起。它的两个与非门用于地址解码逻辑，允许根据 CPU A15 线的状态选择 ROM 或 RAM 芯片，并在系统时钟的低相位期间阻塞 RAM。需要后一个功能，因为地址线不能保证在低相位期间稳定，并且可能导致对随机存储器位置的写入。

其余两个与非门连接成 RS 触发器，以实现串行输出。这是必要的，因为 CPU 无法在多个时钟周期内保持其输出处于同一状态，而串行端口则需要这种状态。相反，[c0pperdragon]使用通常用于实现多处理器系统的 MLB 引脚来产生双时钟脉冲，并根据需要将状态存储在触发器中。然后，可以使用一些定时软件例程来发送和接收串行数据，而无需任何其他硬件。

目前，这个系统的唯一软件是一个简单的演示，它发送回从其串行端口接收的数据，但是如果你喜欢挑战，你可以编写程序来做几乎任何事情。你可能会从其他极简的 6502 板或者在 FPGA 中模拟完整主板的项目中找到一些灵感。
