# 为正义而战

> 原文:[https://hackaday.com/2022/07/16/fighting-the-good-fight/](https://hackaday.com/2022/07/16/fighting-the-good-fight/)

我们在 Hackaday 是开源的超级支持者。软件、硬件或固件，我们希望能够看到它，从中学习，修改它，并自己制作它。这其中有些是自私的，因为当我们不能*看到*它是如何做的，我们就不能*向*你展示它是如何做的。但它也来自一个更深的地方:相信世界会因为分享和开放而变得更美好。

我家里至少有三台设备正在运行开源固件，其中一个是[grbl](https://github.com/gnea/grbl)——它是一个超级简单、超级可靠的 g 代码解释器和步进电机控制器，经受住了时间的考验。这也是 [GPL3 许可](https://github.com/gnea/grbl/blob/master/COPYING)，这意味着如果你想在你的项目中使用代码，并且你修改它以匹配你的特定机器，你必须让那些购买机器的人可以使用修改的版本来修改他们自己。

所以当诺伯特·海因茨注意到[奥图尔的激光雕刻师在运行 grbl 而没有公开代码](https://hackaday.com/2022/07/15/watch-out-for-lasercutter-manufacturers-violating-gpl/)时，他给他们写了一封信。他们以“商业秘密”回应，他再次告知他们的责任，他们仍然没有遵守。[于是他制作了一段视频解释当时的情况](https://hackaday.com/2022/07/15/watch-out-for-lasercutter-manufacturers-violating-gpl/)。

好消息来了！诺伯特在评论中写道，自从帖子被黑客攻击后，他们已经注意到了奥图尔，并与他取得了联系。假设他们正在做正确的事情，这对 grbl 和 Ortur 用户来说都是一个不错的胜利。

在自由软件的世界里，我们都知道“自由”有许多含义，但是我敢打赌，你不必走出我们的社区很远就能找到不知道“自由”软件可以有严格的使用限制的人。(也可能不是——这完全取决于软件作者选择的许可证。)阅读软件许可证是一件糟糕的工作，不管怎么说，最好留给律师而不是黑客去做，我已经数不清自己不浏览一下就点击了多少次 EULA。

所以诺伯特做的是一件好事——教育一个使用 GPL 软件的公司他们的义务。我的直觉告诉我，奥图尔不知道他们需要做什么来遵守许可证，诺贝特告诉他们，即使这需要一些公开的压力。但是现在，Ortur 有机会取得成功，各地的黑客都可以定制驱动他们的激光雕刻机的固件。呜！

现在宣布胜利可能还为时过早，但你可以考虑自己以诺伯特为榜样。虽然如果你不是版权所有者，你不能提起诉讼，但你仍然可以简单地通过礼貌地向那些可能不知道自己违法的公司解释，来捍卫你的自由软件权利。当他们到来时，确保你欢迎他们加入全球开源思想，因为我们都是赢家。我们中的一员！
