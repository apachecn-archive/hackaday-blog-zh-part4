# Solid 能拯救互联网吗？

> 原文：<https://hackaday.com/2020/04/04/can-solid-save-the-internet/>

本周我们在 Solid 上发表了一篇文章，这个项目旨在改变我们今天使用的互联网的隐私和安全方面。蒂姆·伯纳斯·李爵士，这个在工作中作为业余项目发明了万维网的家伙，是它的幕后推手，它有很多值得推荐的地方。我当然希望他们成功。

基本的想法是，取代将你的照片、内容和想法交给社交媒体和其他共享平台，你可以将你自己的个人数据存储在个人在线数据(POD)容器中，并授予这些公司可撤销的访问权限，以代表你访问你的数据。这就像是你自己的网站内容，但是有一个 API 可以在其他地方分享部分内容。

这是一个聪明的合法手段，因为今天你放弃了对你数据的权利，这样脸书公司就可以以你的名义展示这些数据。这给了他们所有的讨价还价的权力，并锁定你到他们的服务。相反，如果你只是给脸书一个可撤销的访问令牌，权力的动态变化。今天，你可以迁移你的数据，删除你的脸书账户，但这是一个很少有人承担的大麻烦。

迈克和我在本周的播客中讨论了这个问题，我们考虑了豆荚的隐私问题。特别是，无论你用哪家公司在社交上分享你的东西，它仍然能够窥探你，描绘你的行为，并以你为广告和其他内容的目标，因为它们在传输过程中就能看到。但是我没能把两者结合起来。

共享你的内容/数据的通用 API 的真正力量在于，它将使从一个共享平台切换到另一个平台变得更加容易。这意味着您可以轻松地迁移到尊重您隐私的系统。如果我们幸运的话，我们会在这个领域看到竞争。同时，存储和托管数据也将是可移植的，有望促进提供者的最佳实践。在你的数据存储在哪里以及如何提供这些数据方面的真正竞争可能会拯救互联网。(或者至少我们可以做梦。)
