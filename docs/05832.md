# 物联网陷阱

> 原文:[https://hackaday.com/2020/02/29/the-iot-trap/](https://hackaday.com/2020/02/29/the-iot-trap/)

我相信你已经听说了 Sonos 扬声器的失败。(如果不是，[在 Hackaday](https://hackaday.com/2020/02/24/ethics-whiplash-as-sonos-tries-every-possible-wrong-way-to-handle-iot-right/) 上了解一下。)基本上，一家销售高级联网扬声器的公司想要淘汰一个旧产品线，并向愿意以旧扬声器“换购”新扬声器的人提供 30%的折扣。问题是:他们并没有真的把它们卖掉，而是更新了一个“自毁”固件，然后拿去回收。

自然，Sonos 最忠实的客户对故意封杀他们忠实的设备很不高兴，随之而来的是一片喧嚣，最终这位首席执行官改变了方向，吃了乌鸦。Hackaday 自己的 Gerrit Coetzee 写了我们的报道，并提到 Sonos 可能无法再为旧产品提供支持服务，并且不想让它们继续存在。以至于它值他们当前产品成本的 30%来摆脱隐性合同。

[![](../Images/bd2c93b24953b7696a9471c3df66b112.png)](https://hackaday.com/2020/02/24/ethics-whiplash-as-sonos-tries-every-possible-wrong-way-to-handle-iot-right/) 通过购买这些物联网设备中的一个，你预先支付了更多的钱，因为该公司承诺在未来将继续支持它所依赖的服务。但是提供这种服务是要花钱的，随着越来越多的“产品”实际上是伪装的服务，我们已经看到一个又一个工作机器关闭的案例，因为公司不想继续为服务付费。无论这家公司是像 Sonos 这样的小公司，还是像谷歌(T4)这样非常富有的垄断者，都无关紧要。不知何故，设计这些产品的人比他们的客户想到的寿命要短得多，并且未能使前期价格覆盖成本。

这让这些公司处境艰难。客户越喜欢这个设备，他们就越想让它运行得更久，当公司最终不得不设法摆脱“终身”合同时，反作用就越大。而且他们正在疏远他们最忠实的客户——那些想让他们的插件运行更长时间的人。鉴于这整个商业模式是全新的，一些公司会犯错误也就不足为奇了。令我惊讶的是，有多少人落入了物联网陷阱。

因此，作为消费者，请将此作为一个警示。如果你所在的公司提供的产品依赖于一项服务来继续发挥作用，问问你自己，你是否真的能够支持客户对产品生命周期的想法。五年后看起来很不错的事情可能会让你的公司在十年后破产。您或您的客户愿意扔掉他们的设备吗？他们应该吗？
