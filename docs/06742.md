# DMCA Vs 黑客

> 原文:[https://hackaday.com/2020/05/30/dmca-vs-hacker/](https://hackaday.com/2020/05/30/dmca-vs-hacker/)

本周报道了一次黑客攻击引起的大混乱，你可能在 Hackaday 上读到过:[中微子] [把一个有机发光二极管屏幕和一个 ESP32 塞进了一个卡西欧计算器](https://hackaday.com/2020/05/07/21st-century-cheating-wifi-in-a-calculator/)。反假冒组织[REACT](https://www.react.org/solutions/online-enforcement/)代表卡西欧在各地提起 DMCA 版权诉讼，包括 GitHub 和 YouTube，并且【中微子】项目的所有痕迹都被[从网上抹去](https://hackaday.com/2020/05/24/dmca-takedown-issued-over-casio-code-that-wasnt/)。

DMCA 是一项有趣的立法。它被用来阻止人们在他们的拖拉机上工作，补充打印机墨盒，并压制不同意见，但在某种意义上，它也让我们拥有我们知道并热爱的互联网。

特别是,“安全港”条款免除了 YouTube 和 GitHub 等在线平台对其托管内容的责任，只要它们在有人对其提出版权主张时删除这些内容。因此，如果内容所有者，比如卡西欧，对[中微子]的 GitHub 和 YouTube 内容发布下架通知，他们必须遵守。如果他认为这个请求是错误的，[中微子]可以提交一份反通知。十到十四天后，假设没有采取正式的法律行动，内容必须恢复。([参见第 512 节(g)](https://www.copyright.gov/title17/92chap5.html#512) 。)

撤下通知和反通知都是有约束力的法律文件，并宣誓作伪证。通知和反通知可以被使用或滥用，而众所周知，版权法充满灰色地带。GitHub 的好处在于，他们会公布他们收到的所有 DMCA 通知和反通知，所以[这里是](https://github.com/github/dmca/blob/master/2020/05/2020-05-20-Casio.md)让你自己判断。

由于伪证罪的影响，我们不能说 REACT 的人谁提交了撤销*明知*提交了一个虚假的请求*恶意*——这将指控他们违反联邦法律——但我们可以肯定地说，看起来他们在这方面大错特错。他们肯定不是编码员。

好消息是代码在 GitHub 上恢复了，但奇怪的是描述黑客的视频在 YouTube 上仍然不见了。

但这是卡西欧如何看待和反应的:他们看到了他们产品的不光彩之处——它可能被用于学校作弊——他们派出了法律攻击队。如果是这样，那就烂了。
