# 嵌入式 Rust Hack 聊天

> 原文:[https://hackaday.com/2021/05/10/embedded-rust-hack-chat/](https://hackaday.com/2021/05/10/embedded-rust-hack-chat/)

加入我们 5 月 12 日(星期三)中午与 James Munns 的[嵌入式 Rust Hack 聊天](https://hackaday.io/event/179265-embedded-rust-hack-chat)！

像时尚一样，编程语言在很大程度上取决于个人品味。专业开发人员通常对给定项目使用哪种语言没有太多发言权，无论是出于遗产还是团队原因，但是如果他们有选择的话，他们可能会选择最适合他们思维方式的语言。有些语言只是比其他语言更“适合”不同的大脑，当语言和开发人员之间的一切都同步时，代码似乎可以毫不费力地通过键盘流到屏幕上。

Rust 是一种一直高居开发者“最爱”列表榜首的语言。对于一门以个人项目开始，并且只存在了十几年的语言来说，这真的很有意义。Rust 对安全性和性能的强调可能与此有很大关系。由于其安全的并发性、内存安全性以及与 C 和其他语言的互操作性，Rust 已经在嵌入式开发社区中取得了相当大的进展。

为了了解更多关于嵌入式系统中 Rust 的知识，James Munns 将在 Hack Chat 上停下来。James 是一名嵌入式系统工程师，拥有为各种系统开发软件的历史，包括安全关键型航空电子设备和快速原型物联网系统。他是 [Rust 嵌入式工作组](https://t.co/7WIyOzQrh7?amp=1) 的创始成员，也是 [亚铁系统](https://t.co/pLWNR2yBhS?amp=1) 的创始人，这是一家专注于 Rust 系统开发的咨询公司，专门从事嵌入式系统开发。詹姆斯也曾经为 Hackaday 写文章，所以他一定是个很酷的人。因此，请访问 Hack Chat，了解 Rust 可能在哪些方面帮助您完成下一个嵌入式项目。

[![join-hack-chat](../Images/cff5733f39a173302e1cd102e75ac107.png)](https://hackaday.io/messages/room/2369) 我们的黑客聊天是 [Hackaday.io 黑客聊天群发消息](https://hackaday.io/messages/room/2369)中的社区直播活动。本周，我们将于太平洋时间 5 月 12 日星期三中午 12:00 坐下来讨论。如果时区束缚了你，我们有一个[方便的时区转换器](https://www.timeanddate.com/countdown/generic?iso=20210512T12&p0=224&msg=Embedded+Rust+Hack+Chat&font=cursive)。

点击右边的那个发言气泡，你会被直接带到 Hackaday.io 上的黑客聊天群，不用等到周三；随时加入，你可以看到社区在谈论什么。
