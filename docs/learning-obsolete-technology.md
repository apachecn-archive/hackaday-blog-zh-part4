# 学习过时的技术

> 原文：<https://hackaday.com/2022/05/07/learning-obsolete-technology/>

汤姆·纳尔迪和我在播客上谈论他的复古电脑节之旅，他承认在他第一次旅行之前并不是一个复古电脑爱好者。但他最终将一些二进制机器代码键入到一些古老的硅集合中，并得到了它。在同一集中，本周的声音是一个[史端乔开关](https://hackaday.com/2022/04/14/the-simplest-electro-mechanical-telephone-exchange-that-actually-works/)——旧电话交换中心的旧机电“大脑”。我用的样本[来自 YouTube](https://www.youtube.com/watch?v=BvhqZbtU_Zs) 上的“看起来像妈妈没有电脑”的山姆，他为他的博物馆弄了一台电脑，他认为这太棒了。

为什么会有人喜欢这种老(过时？)tech？这当然不是因为它具有压倒性的能力——巨大的旧交换机很容易被一堆硅片取代，甚至不要让我开始使用汤姆正在使用的旧 blinkenlights 计算机。在这两种情况下，人们都比他们正在玩的技术年轻得多，所以这排除了怀旧的可能。还剩下什么？

[![](img/39b77190b4a99d1fc17b475e2041430f.png)](https://hackaday.com/wp-content/uploads/2022/05/vcfeast22_gallery8.jpg) 我认为有时候旧技术更直接，更容易理解，更切实，能引起人们的共鸣。在这个时代，我们都有可以做任何事情的神奇设备，用令人愉快的语言编程，使用不亚于魔法的库来使困难的事情变得简单，理解事物如何实际工作是一种罕见的商品。

但从技术上来说，我们发现自己处于一个奇怪的位置，在一个设备的有用性和功能性与从根本上理解其工作原理的能力之间几乎必然存在权衡。
