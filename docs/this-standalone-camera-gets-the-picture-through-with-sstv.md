# 这款独立相机通过 SSTV 拍摄照片

> 原文：<https://hackaday.com/2022/11/20/this-standalone-camera-gets-the-picture-through-with-sstv/>

如今，给别人发一张照片就像拿出智能手机，通过电子邮件或短信发送一样简单。这很简单，一个孩子都能做到，但这种简单的用户体验掩盖了巨大的复杂性，从手机中的压缩算法到将手机连接在一起所需的大量分布式基础设施。尽管所有这些基础设施都很棒，很有用，但有时对工作来说太多了。

[Dzl TheEvilGenius]似乎就是这种情况，他只是想从远程位置发送低分辨率图像。事实证明，大约 70 年前，hams 用慢扫描电视或 SSTV 解决了这个问题。当世界上大多数人在普通电视上的“我爱露西”前安顿下来时，业余无线电操作员正在想办法如何使用他们的设备向世界各地发送图片。但是以前的 hams 不得不在这个问题上投入大量的设备，[Dzl]只是使用了一个带有摄像头和一些自定义代码的 ESP-32 来处理图像。MCU 的一个 GPIO 引脚的输出是 PWM 音频信号，可以直接馈入廉价便携式收发器的麦克风输入。

为了解码信号，[Dzl]使用了许多可用的 SSTV 程序中的一个。没有提到接收器，尽管它可能是从另一个宝丰到 SDR 加密狗的任何东西。如果您只是想研究一下等式的接收和解码方面，本文中提供了代码，以及一个编码图像的音频文件。

我们可以看到这样的东西为远程安全摄像头工作，甚至为侦察狩猎点工作。如果你想复制这一点，请记住，如果你想在火腿波段上传输，你需要一个许可证——放松，[这很容易](https://hackaday.com/2019/03/08/the-50-ham-getting-your-ticket-punched/)。
