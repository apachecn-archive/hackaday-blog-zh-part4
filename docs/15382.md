# 一次听 64 MHz

> 原文：<https://hackaday.com/2022/11/16/listen-to-64-mhz-at-once/>

我们设想，如果[技术头脑]告诉我们他在听高频波段，我们可能会问他是哪一个？他的回答可能只是“所有人”这要归功于他评论的 RX-888 MKII SDR，它在无线电频谱上提供了 64 MHz 的窗口。你可以看到下面的视频评论。

这些不是特别便宜，但有了这样的带宽和 16 位分辨率，如果你需要这样的马力，它是值得的。有一个单独的输入 VHF 信号 64-1700 MHz 的带宽只有 10 MHz，但仍然。

当然，为这样的东西制作一个非常宽带的前端是很重要的，所以我们想知道其性能如何与带宽更少的类似价格的单元相比较。另一方面，它在视频中似乎确实足够好。使用的软件将测试限制在 32 MHz 的带宽内，这已经足够了。

说到软件，我们注意到 SatDump 和 SDR++的开发者对 RX-888 的软件的[状态并不满意。我们不确定这是否仍然是一个问题，但该设备似乎至少在视频上工作良好。](https://docs.google.com/document/d/1hG634yVkBBIMykmmnosUxw7gumAoc3z1SmWDrH1TucM/edit#)

谈到更高端的特别提款权，现在有很多选择。我们喜欢冥王星的发射和接收。当然， [RTL-SDR](https://hackaday.com/2019/07/31/rtl-sdr-seven-years-later/) 有点像是从业余爱好 SDR 开始的，但你不能指望其中一个有那么多带宽。
