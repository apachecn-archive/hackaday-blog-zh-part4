# 社会工程和菜单

> 原文:[https://hack aday . com/2021/07/01/social-engineering-and-menus/](https://hackaday.com/2021/07/01/social-engineering-and-menus/)

如果你遵循网络安全黑客的方法——或者只是看着*机器人先生*——你可能知道获得某人密码的最好方法是询问。当然，你可能不能只说“嗨，我是个坏人。能给我你的密码吗？”但是，你可以使用各种各样的技巧，比如假装在这个人的 IT 部门，或者在管理层，或者通过编造一个危机来克服他们更好的判断力。但是当然，作为聪明的计算机人，我们对这种事情是免疫的，对吗？我们的武器库中也不需要这些花样。

这是真的吗？令人惊讶的是，无论我们是谁，有多少微妙的事情影响着我们认为是理性的决定。考虑去餐馆吃饭。简单吧？你看菜单，选择你想要的，然后点菜。没有人在影响你。但是他们是。根据 BBC 的一篇文章中的[，T2 有一整个行业的菜单“工程”](https://www.bbc.com/future/article/20171120-the-secret-tricks-hidden-inside-restaurant-menus)会想出如何让你点昂贵的食物。

你可能不认为菜单的社会工程对我们来说是一项伟大的技能。但是也许你的新开源项目需要合作者。也许你的创业公司需要投资人。也许你希望有人看看你的简历。也许对用餐者起作用的同样的技巧在那些情况下也会起作用。

## 不仅仅是菜单

不仅仅是菜单。杂货店有一整套科学知识，告诉你在哪里放牛奶之类的东西，以确保你有机会购买其他东西。他们也知道什么地方卖东西。赌场也是明智的。引人注目的老虎机往往比藏在角落里的老虎机支付更多。除非他们在高利润的桌上游戏中可见，否则他们可能会惹恼大赌客。

[![](../Images/87ed8eb7ebc17ebf0d6d7458ac5945fd.png)](https://hackaday.com/wp-content/uploads/2021/05/pop.png) 电视节目*智力游戏*做了一个关于“诱饵效应”的实验他们让电影观众选择 3 美元一小份爆米花或 7 美元一大份爆米花。他们几乎不卖大爆米花，当被问到时，顾客抱怨大爆米花的价格。后来，他们给类似的顾客提供了三种选择。小号和大号还是一样的，但是他们包括一个 6.5 美元的中号。每个人都想要中号的，但后来意识到只需多花 50 英镑就能买到大号的，于是就这么做了。菜单选项呈现的方式将结果从小到大移动，消费者对此一无所知。

## 回到您的常规预定菜单

正如爆米花实验所显示的，我们对菜单定价的敏感性是无法解释的。菜单上使用较长单词的菜肴往往价格更高，食客也不会介意。餐馆还会把最贵的食物放在最上面，这样当你吃到价格较低的食物时，它们看起来更合理。

那么是什么吸引了食客呢？像字体和颜色这样简单的东西显然可以带来巨大的变化。斜体字传达了质量，矛盾的是，华丽的字体可能很难阅读，但可能会让东西看起来味道更好。显然，标有难以辨认字体的葡萄酒比装在更易辨认的瓶子里的同样的葡萄酒得分更高。

研究表明，人们将圆形字体与甜食联系在一起，将棱角分明的字体与咸食联系在一起。说不通，但显然是真的。

## 介绍会；展示会

显然，薯条没有“厚脆薯条”卖得好。佩恩和特勒的秀，*扯淡！*有一集，食客在一家高档餐厅吃到难吃的食物，他们很喜欢，因为服务员什么都卖。例如，商店品牌的搅打浇头是手工搅打的慕斯。大多数人吃着廉价的街头小吃，就好像这是一顿五星级的大餐。

菜单专家声称，描述性语言可以增加 27%的销售额。斯坦福大学发现，“甜甜的咝咝作响的四季豆”比“四季豆”多点了 23%。这是有道理的，但奇怪的是，即使你知道这是夸张的，它仍然影响着你。

想想麦当劳四分之一磅汉堡的描述:

> …当你点的时候，热腾腾、美味多汁、煮熟了。它用少量的盐和胡椒调味，在扁平的铁烤架上咝咝作响，然后在芝麻小面包上浇上切成薄片的洋葱、扑鼻的泡菜和两片融化的美国奶酪。

一个扁平的铁烤架？其他地方有没有不扑鼻的泡菜或者在热汉堡上不融化的奶酪？麦当劳的厨师是不是在你的汉堡里加了点调料？但它仍然有效。你可能正在用手机订购。

图片和视频是一个大杂烩。有时食客会把食物图片和低质量联系在一起。此外，还有一种担心是，你在照片中看到的食物——正如你在下面的视频中看到的那样，这通常不是真正的食物，因为食物很难拍得很好——看起来会比你餐桌上的食物更好。如果你在快餐店吃过饭，你就会知道这是完全可能的。

## 那又怎样？

你可能认为这种社会工程只是为店主或餐馆老板准备的。但是有多少次你会选择 Chrome 这样的工具而不是 Firefox 这样的同类工具呢？如果你想让人们使用你的作品，也许需要一些社会工程。

我们都见过伟大的项目和公司步履蹒跚，而较小的项目和公司却欣欣向荣。不用指名道姓，操作系统、Linux 发行版、编辑器、编程语言、视频格式都见过这种效果。因此，当你编写下一个伟大的物联网库时，也许它不应该是“让你使用网络请求控制设备的功能。”也许它应该是“一个强大而安全的图书馆，让您可以轻松地从世界任何地方控制设备，简化并提升您的高科技生活方式。”当然，这听起来很荒谬，但它被证明是有效的。
