# 烘烤它来重塑它:煮旧磁带来恢复数据

> 原文:[https://hackaday . com/2021/08/29/bake-it-to-re make-it-cooking-old-magnetic-tape-to-recover-data/](https://hackaday.com/2021/08/29/bake-it-to-remake-it-cooking-old-magnetic-tape-to-recover-data/)

我们这些年纪够大的人可能还记得第一次出现的文本冒险游戏类型的全盛时期。总部位于伦敦的 Magnetic Scrolls 是一家早期的先锋公司，为首批 Amiga 和 Atari ST 平台生产游戏。快进到 2017 年，Magnetic Scrolls 的原始联合创始人和核心开发者[Hugh Steers]发起了一项倡议，要在现代平台上复兴和重新发布原始游戏。自从 20 世纪 80 年代最初用于开发的 DEC MicroVAX 在复古计算界并不罕见，并且在某人的阁楼空间发现了包含源代码的媒体，复兴游戏并不是一个高要求。

首先，他需要从备份磁带中恢复原始源代码的副本。但有一个问题，原来腐烂的磁带使用了一种不稳定的聚氨酯基粘合剂来将氧化物材料(存储数据的材料)粘到背带上，这种粘合剂可以在多年后吸收水分。

直到你试着读磁带，然后你被所谓的[粘棚综合症](https://en.wikipedia.org/wiki/Sticky-shed_syndrome)绊倒，才会发生什么。其次，你可能会发现少量的氧化层从磁带上脱落，覆盖在复杂的磁带机构内的读取头、滚轮和导轨上。这很快导致它粘起来，卡住，潜在地咀嚼磁带，并永久地破坏它。

DEC TK50Z 磁带机的行为进一步加剧了这种情况，因为作为正常操作的一部分，它需要穿梭整个长度的磁带。

一个临时解决方案是在烤箱中烘烤磁带，以驱除湿气并降低粘性，使其能够安全地通过驱动器。那么只剩下氧化物脱落的问题了。TK50Z 驱动器被替换为 TZ30，TZ30 减少了磁带的穿梭，但关键是通过简单的黑客攻击，可以在读遍之间用 IPA 清洁磁头。这足以防止出错，并允许从磁带中读取足够的数据，以恢复几个游戏的代码，为重新发布过程做好准备。

休息后的视频显示[Rob Jarratt]正在完成数据恢复过程。
