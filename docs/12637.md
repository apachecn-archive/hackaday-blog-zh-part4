# 代码错误:扩展你的思维

> 原文：<https://hackaday.com/2022/01/29/code-wrong-expand-your-mind/>

用“错误”的方式做事情的真正好处是有太多的变化了！如果你以正确的方式，最快的方式，或者最佳的方式做某事，那么，只有一种方式。但是如果你做错了，你有更多的设计空间。

[![](img/01ef2b6edf63c594da50b050a19bec12.png)](https://hackaday.com/wp-content/uploads/2022/01/Piet_Hello_World.gif) 例子:[深奥的编程语言](https://hackaday.com/2022/01/26/strange-computer-languages-a-hackers-field-guide/)。品种之多令人惊叹。有些语言*旨在*让人读不懂，或者听起来像莎士比亚的十四行诗，或者烹饪食谱，或者摇滚民谣。一些最早的深奥的语言只是笑话:当时“真正”编程语言的所有麻烦的汇编，但仍然可以工作。有些用彩色像素网格来表示指令。有些用一种相当于加密的方式来表示代码，对它们进行编程的唯一方法是强行占用代码空间。其他人，包括臭名昭著的 Brainf*ck，实际上没有他们说唱的一半糟糕——这是图灵机的一个非常直接的实现。

所以你有一套语言，它们被设计成最大程度地不同于其他语言，或者传统的编程语言，但仍然能够指导计算机做你想做的事情。如果你眯起眼睛，尽可能多地把它们放在一起看，从这些古怪语言的混杂中浮现出来的就是计算的本质。每种语言都尽量不出错，所以它们的共同点只能是编码不可避免的核心。

虽然对比 Java 和 C++，或 Python 可能很有趣，但几乎每一种严肃的编程语言都有如此多的共同点，以至于没有指导意义。他们都做得很对，这意味着他们主要是考虑到人的因素。打哈欠。要真正搞清楚计算的基础是什么，你必须把它搞错。
