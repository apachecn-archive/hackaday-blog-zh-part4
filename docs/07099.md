# 编码的理由

> 原文:[https://hackaday.com/2020/07/04/a-reason-to-code/](https://hackaday.com/2020/07/04/a-reason-to-code/)

我的儿子刚刚到了让他成为所有学习编码玩具的靶子的年龄。不可否认的是，我们已经看到了一些类似于[标志的](https://en.wikipedia.org/wiki/Logo_(programming_language))玩具，你可以指示一个海龟机器人做一些动作，然后重复这些动作。毕竟，如果将一个问题分解成子问题并自动重复不是编程的本质，我不知道什么才是。

但事实是:我认为绘画机器人比他更酷。如果你问一个孩子“嘿，你想要一辆会画画的车吗？”这在机器人列表中是相当低的。我并不是说一旦他掌握了更多的代码，他就不会投入进去，他可以开始让它做有趣的事情，但是就其本身而言，画画并不那么令人印象深刻。他画得很好，非常感谢。

[![](../Images/76deab4bebdb427682e5506a28c097ff.png)](https://hackaday.com/wp-content/uploads/2020/07/DSCF1741_thumbnail.png) 与此同时，我正在制作一个机器人手臂。或者更确切地说，我开始着手另一个永远不会完成的机器人手臂。(坦白地说，我不知道我会用机器人手臂*做*什么。)但至少我是从手爪和手腕开始的。对一个孩子来说这很酷，但是编程太复杂了。所以我把大脑取出来，把伺服系统接到遥控飞机遥控器上。只是晃动着这个用胶带粘在桌子上的东西，就让他上瘾了。这个周末，我们正在建造一个遥控摘樱桃的手臂，把它放在杆子上，因为樱桃正在上市。他的主意！

所以不用编码。在我看来，他还是有点太年轻了。但是像这些储存在他潜意识深处的愚蠢的小项目，会给他一个*理由*在未来编程，会让他明白知道如何编程是有用的。现在我所需要的是一个完成机器人手臂项目的理由…
