# 裸机 STM32 编程 Epic 指南

> 原文：<https://hackaday.com/2022/12/05/epic-guide-to-bare-metal-stm32-programming/>

[Sergey Lyubka]整理了这份裸机微控制器编程的[史诗指南](https://github.com/cpq/bare-metal-programming-guide)。虽然一般概念应该适用于大多数微控制器，但[Sergey]的示例特别涉及采用基于 ARM 的 STM32F429 微控制器的 Nucleo-F429ZI 开发板。

在计算机系统领域，裸机编程通常是指在没有操作系统介入的情况下对处理器进行编程。这通常适用于编程 BIOS、硬件驱动程序、通信驱动程序、操作系统元素等等。即使在嵌入式编程的世界中，事情通常是相当低级的(接近金属)，我们已经习惯了大量的硬件抽象。例如，我们经常在各种库、引导加载程序和集成开发工具的基础上开始项目。

当我们放弃这些抽象概念，直接在微处理器或微控制器上编程时，我们是在裸机上工作。[Sergey]恰当地将此定义为“仅使用编译器和数据表，别无其他”对微控制器进行编程他的指南从检查处理器的存储器映射和寄存器开始，包括存储器映射 I/O 引脚和其他外设的位置。

该指南指导我们编写一个最小的固件程序，从引导向量到连接到 I/O 引脚的 LED 闪烁。演示将继续安装和使用必要的工具，如编译器、链接器和 flasher。我们继续讨论越来越高级的主题，如定时器、中断、UART 输出、调试器，甚至配置嵌入式 web 服务器以展示完整的设备仪表板。

虽然最初更耗时，但靠近金属工作提供了对硬件操作的大量额外洞察和控制。关于这个主题的更多信息，你可能会喜欢我们关于裸机 STM32 编程的 [STM32 Bootcamp](https://hackaday.com/2020/11/17/bare-metal-stm32-from-power-up-to-hello-world/) 系列。
