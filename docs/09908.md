# 流言终结者 Tidal 的 MQA 格式——如何衡量？

> 原文:[https://hack aday . com/2021/04/21/myth bushing-tidals-mqa-format-how-it-measure-up/](https://hackaday.com/2021/04/21/mythbusting-tidals-mqa-format-how-does-it-measure-up/)

MQA 是一种音频格式，声称使用独特的“折纸”算法，承诺比其他格式更好的质量和音乐性。有时，它被声称是一种无损格式，并被流媒体服务称赞为高保真音乐的最终格式。由于这种格式是闭源的，编码器也不公开，这些说法很难验证。然而，[金声]不是昨天才诞生的，它不择手段地开始测试 MQA。结果令人担忧。(视频，下面嵌入。)

要真正测试这种格式，唯一简单的方法是将音乐发布到使用这种格式的 Tidal 流媒体服务上。[GoldenSound]走这条路，试图得到一些测试文件发表。当出版公司报告说 MQA 软件“不能对文件进行编码”时，这就碰壁了。变通方法？[GoldenSound]只需将一些音频测试内容剪切到原声音轨的中间，然后重新提交文件，文件就被接受了，没有进一步的抱怨。

通过对从 Tidal 上下载的内容进行测试，[GoldenSound]发现了关于 MQA 的说法不成立的证据。在 MQA 处理的文件中经常会发现大量的附加噪声，从 Tidal 提供的文件显然不是无损的。此外，MQA 的“蓝光”认证系统，旨在向听众保证他们正在听一个相同的工作室发布，被证明是误导，如果不完全是假的。

在给 MQA 写信要求对他的发现做出回应后，[GoldenSound]的测试文件很快就从 Tidal 上删除了。该公司最终对视频中讨论的一些发现提出了质疑。总的结论是，如果没有公开、透明的工具来分析这种格式的性能，就不可能验证该公司的说法。
