# RF 工程师的 FPGA 学习之旅

> 原文:[https://hack aday . com/2020/03/12/an-RF-engineers-FPGA-learning-journey/](https://hackaday.com/2020/03/12/an-rf-engineers-fpga-learning-journey/)

[KF5N]承认自己不是数字设计工程师；他更喜欢模拟射频方面的东西。但他最近接了一个项目，在 Ubuntu box 和 Intel MAX10 FPGA 之间进行通信。他在最近的一次业余无线电会议上做了一个报告，讲述了他所学到的东西以及如何开始。

视频里说了很多关于英特尔(曾经是 Altera)的事情。然而，休息后近 40 分钟的视频不是一步一步的教程，所以即使你对其他设备感兴趣，你也可能会喜欢看它。如果你只编程过一个 FPGA，这个视频可能不会引起你的兴趣——你不是目标观众。然而，在大约 00:31 时，他确实推荐了一些书籍和一些非常便宜的 FPGA 板，所以这并不完全是浪费。

[KF5N]讲述什么是 FPGA，以及它与微控制器有何不同。他还推荐康奈尔大学的(布鲁斯·兰德的)课程材料。他对自己尝试的在线课程不太感兴趣。当然，由于他使用的是英特尔芯片，他还推荐了英特尔课程。很多视频介绍了如何节省购买开发板的成本。康奈尔大学的课程需要一个 250 美元的非常强大的主板。那也很贵，所以他推荐了一个 85 美元左右的更轻的版本。

他还谈到了工具链和他的 Linux 机器接口项目。他最终获得了一个高达 30 MHz 的 SPI 接口。他还谈到使用 Julia 构建一个驱动程序来与 PC 端的接口对话。

我们没有注意到他提到我们自己的 [FPGA bootcamp](https://hackaday.com/2018/08/06/learn-fpga-fast-with-hackadays-fpga-boot-camp/) ，尽管他确实提到了 Hackaday.io 上的项目。如果你想看[一个类似的视频，但使用了开源工具](https://hackaday.com/2019/12/06/david-williams-is-fpga-curious/),【David Williams】在 Superconference 上做了一个演讲，提供了同样的概述，但使用了 Yosys 和其他相关工具。
