# 远程视频:了解软件定义无线电(SDR)的基础知识

> 原文：<https://hackaday.com/2021/01/19/remoticon-video-learning-the-basics-of-software-defined-radio-sdr/>

你涉足 SDR 的海洋了吗？虽然黑客软件定义无线电多年来一直是一个热门话题，但第一次尝试它可能会有点令人生畏。下面是 Josh Conway 在 2020 Hackaday Remoticon 期间举办的 SDR 概述研讨会为您带来的变化。

Josh 的演讲首先简单介绍了 SDR 的定义，然后概述了现有的硬件和软件。收音机的硬件设计可能很容易构建，但它们将限于单一协议，例如，调频收音机不能监听 433 Mhz 无线门铃。SDR 突破了这一限制，转而采用一种无线电硬件，只需修改控制它的软件，就可以重新配置这种硬件，使之与协议一起工作。这使得无线电硬件更加昂贵，但也意味着你可以收听(有时发送)各种设备，如无线门铃或汽车轮胎压力传感器，以及基于无线电的基础设施，如飞机转发器和气象卫星。

这是您想要的快速入门，因为它以恰到好处的深度解释了许多主题。硬件概述包括 RTL-SDR、ADALM-PLUTO、HackRF、KerberosSDR 和 BladeRF(我们刚刚在周末的 WiFi 协议上介绍了这些协议)。软件方面，Josh recaps GQRX、SDR#、SDRAngel、ShinySDR、通用无线电黑客、Inspectrum、SigDigger、RPITX、GnuRadio Companion、REDHAWK。在回答研讨会与会者的问题之前，他还向我们介绍了各种天线类型。

如果 SDR 仍然不在您的工具箱中，那么现在是重新审视它的好时机。一旦你通过了“你好，世界”这个阶段，还有很多可以探索的，比如[我们在另一个 Remoticon 演讲中提到的那些令人敬畏的射频辐射测试技巧](https://hackaday.com/2021/01/08/remoticon-video-basics-of-rf-emissions-debugging-workshop/)。
