# 低于 1000 美元的立方体卫星？

> 原文:[https://hackaday.com/2021/04/28/cubesat-for-under-1000/](https://hackaday.com/2021/04/28/cubesat-for-under-1000/)

想建造自己的立方体卫星，但被价格所阻？工程中可能会有一个解决方案——[RG Sat]挑战自己[设计并建造一个不到 1000 美元的](https://www.youtube.com/watch?v=m8TSiKHZbC8)。(视频，嵌在下面。)

在第一个视频中，他首先对可用的低成本选项进行了调查，发现没有低于 10，000 美元的完整套餐。当你添加了所有必要的“选项”时，最终的总数可能会远远超过 20，000 美元。

他的想法也不仅仅是白日梦。在他开始这个项目后的 15 个月里，[RG Sat]设计并制造了航空电子设备和电力系统电路板，目前正在测试他的太阳跟踪器设计。软件用 Rust 写，只是因为他想学点新东西。如果你想自己构建一个的话，你可以在项目的 GitHub 库上查看硬件设计文件和软件设计文件。

[RG Sat]列出了一个令人信服的案例，但我们想知道是否有一个主要的陷阱潜伏在黑暗的某个地方。事实上，[RG Sat]自己也提出了这个问题，“这些高成本从何而来？”我们的第一直觉是指向空间和/或测试的合格零件。但是，如果你不关心卫星寿命或故障率，那么也许[RG Sat]在这一点上是对的。

然而，退一步来看大图，与发射成本相比，立方体卫星的价格可能只是沧海一粟，除非你免费搭乘。硬件是降低成本的最佳选择吗？无论如何，[RG Sat]的项目一定会提供有趣和有用的结果，无论他成功实现他的目标，还是证实你确实需要 10，000 美元来建造一个立方体卫星。我们将密切关注他的进展。

在之前，我们已经写了关于[开源立方体卫星的文章，也写了一个失败任务](https://hackaday.com/2021/01/28/open-source-cubesats-ease-the-pain-of-building-your-own/)的[事故分析，其中包含了一些很好的教训。感谢[杰里米·格罗斯]的提示。](https://hackaday.com/2020/01/24/lessons-learned-from-a-cubesat-postmortem/)
