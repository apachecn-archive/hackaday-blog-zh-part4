# 魔镜——低 CPU 预算

> 原文：<https://hackaday.com/2022/11/26/magic-mirror-on-a-low-cpu-budget/>

对于相当多的黑客来说，仍然很难找到一款功能相当强大、价格不贵的树莓 Pi。[Rupin Chheda]想要构建一个具有基于网络的前端的魔镜，一个足够现代的 Raspberry Pi 就可以很好地工作。遗憾的是，他所能得到的只是单个 1 GHz 内核 512 MB RAM Zero W 主板，他发现这些主板无法很好地运行 Chromium，更不用说旁边的 web 服务器了。不要放弃，[Rupin]给我们一步一步的讲解如何创建一个显示单个网页的低占用空间的 Raspbian 安装。

从 Raspbian Lite(一个默认情况下不附带任何桌面功能的发行版)开始，他展示了如何为它配备一个最小的 GUI——不需要桌面环境，只需要一个带有 OpenBox 窗口管理器的 X 服务器，因为 kiosk 模式应用程序不需要更多。代替 Chromium，你可以安装美岛莉，这是一个精简的浏览器，在单一网站模式下工作得很好，并且[Rupin]向你展示如何让它自动启动，以及确保你的显示器不会进入睡眠的小窍门。网络服务器运行在 Heroku cloud 上，但是我们打赌，通过这样一个最小的安装，它也可以运行在设备本身上。

有了这些说明，当你只有一个相当基本的 Raspberry Pi 板时，你可以很容易地构建一个低功耗的单页浏览器。当然，到目前为止，魔镜是一个研究充分的话题，但是你总是可以在一个老话题上放一个新的旋转，就像在这个基于复古电视的建筑中。你也不需要建造一面魔镜来使用这个黑客技术——建造[一个食谱亭！](https://hackaday.com/2016/01/23/pi-zero-recipe-kiosk/)
