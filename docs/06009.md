# GitHub 在路上

> 原文:[https://hackaday.com/2020/03/18/github-on-the-go/](https://hackaday.com/2020/03/18/github-on-the-go/)

很难找到不与 GitHub 进行交互的软件开发人员。即使你没有自己的项目，网站上也有很多可以学习和借鉴的东西，几乎无处不在。然而，当你需要 GitHub 的时候，你可能不得不求助于你的手机浏览器，并且体验降低。GitHub for Mobile 现已停止测试，并承诺提供更加流畅的手机 GitHub 体验。

除了处理任务和问题之外，您还可以查看和合并提取请求。该应用程序还会发送手机通知，这非常方便。正如你所料，你可以在各自的商店里买到安卓或 iPhone 版的应用程序。

[Andrew Allbright]在下面的视频中快速回顾了 iOS 版本。令人惊讶的是，GitHub 已经成为开发中不可或缺的一部分，它几乎失去了自己的品牌。有一天有人告诉我们，他们为我们建立了一个“GitHub”来分享一些私人作品。当然，它们意味着私有服务器上的 Git 存储库。很像施乐或 Velcro，对一些人来说，GitHub 就是 Git 的同义词。

自从微软收购 GitHub 以来，GitHub 已经有了很多活动。我们最近讨论了他们如何提供[命令行工具](https://hackaday.com/2020/02/15/github-goes-gui-less/)。如果你需要一个使用 Git 和 GitHub 的借口，看看我们的[愚蠢的 Git 技巧](https://hackaday.com/2017/05/23/stupid-git-tricks/)。
