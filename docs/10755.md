# 开源是一种选择

> 原文:[https://hackaday.com/2021/07/17/open-source-is-choice/](https://hackaday.com/2021/07/17/open-source-is-choice/)

如果你还不了解开源 Audacity 音频编辑软件的许可问题，请花一点时间阅读汤姆·纳尔迪的文章，了解一下最新情况。简而言之，一家盈利性公司购买了该商标和软件，宣布了在没有遥测技术的地方引入遥测技术的计划，对隐私政策进行了不祥的修改，禁止未满法定年龄的人使用该软件，并要求所有以前的开发者默认发布该软件的开源许可证的变化。与此同时，Muse 公司表示，它将保持软件开放，并在遥测问题上摇摆不定。

[![](../Images/e07d0d1151e3cd24c29eba1342f88b18.png)](https://hackaday.com/wp-content/uploads/2021/07/tenacity-logo-dark-readme_thumbnail.png) “胆大妄为”会怎么样？谁知道呢。但是，谁在乎呢？代码库的至少一个分支已经完成，遥测技术被移除，旧的开放许可已经就位。开源最棒的一点是，我一点也不在乎我的软件是叫 Audacity 还是叫 [Tenacity](https://github.com/tenacityteam/tenacity) ，这是我每周用来制作播客的软件。但是因为我没有支付任何许可费，所以用不同的名字下载相同的软件，减去一些反功能，我完全不用花钱。如果开发社区转向 Tenacity，一切都会好的。

汤姆认为 Audacity 品牌太大而不能倒，而缪斯将会在他们手中获得成功。特别是如果他们开始实施新的必备功能，他们可以证明他们所存储的任何计划都是正确的，即使他们只在更严格的许可证下作为“免费增值”的 Audacity Pro 提供。当*和*发生时，我将不得不在这些功能和成本之间做出选择，但只要 Tenacity fork 得到足够的关注，我*不会被冷落。所以最终用户有了更多的选择，对吗？太酷了。*

将此与闭源软件进行比较。在那里，当所有者做出一个不受欢迎的决定时，你只需要接受它或者跳到一个完全不同的软件包。如果你已经擅长使用该软件，这可能会很昂贵，而且在许可和学习之间，有很多不利于转换的因素。在这种情况下并非如此。如果我不想在离线编辑音频时被跟踪，我也不必被跟踪。Woot。

房间里的大象当然是开发和调试社区，现在做预测还为时过早。然而，同样的规则也适用于开发者和用户:在两个几乎相同的代码库之间切换就像`git remote add origin`或`apt get install tenacity`一样简单。(无报酬)开发人员可以在 forks 中自由选择，因为他们喜欢这些条款和条件，因为与一群人一起工作更愉快，或者因为他们喜欢一个标志的颜色多于另一个。用户同样可以自由选择。

时间会证明 Audacity 是否会像僵尸 OpenOffice 一样被下载，尽管 T2 的图书馆比它好得多，只是因为前者的知名度。我知道这种分裂激怒了一些人，尤其是在 LibreOffice 开发社区，而且更好的软件享有更少的声誉*似乎不公平。但对于我们这些了解内情的人来说，这只是更多的选择。这很好，对吗？*
