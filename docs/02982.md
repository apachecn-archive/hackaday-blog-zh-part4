# 全栈网络浏览器

> 原文:[https://hackaday.com/2019/05/14/a-full-stack-web-browser/](https://hackaday.com/2019/05/14/a-full-stack-web-browser/)

面试成为全栈工程师很难。这比申请初级开发人员的工作要难得多，在初级开发人员的工作中，你需要在白板上穿过一棵红黑树。对于全栈工作，他们只是给你一堆 2N2222 晶体管。(第一家公司不是很合适，我最终找到了一个地方，给了我一些 2N2907s 面试。)也就是说，看你能把一些掺杂硅推进多远是一个挑战。恰当的例子是，[阿拉斯泰尔·休伊特]。他正在建造一台电脑，从大门开始浏览万维网。

这个项目的目标是只使用 TTL 逻辑浏览网页。这带来了乍一看并不明显的问题。首先是能够在屏幕上显示文本。现在做到这一点最简单的方法是获得一大堆现代存储器，对于一台 20 世纪 70 年代的老式计算机来说，这些存储器的速度快得惊人。这允许 VGA 输出，是的，我们已经看到了很多使用大内存输出 VGA 的版本。事实证明，这些 RAM 和 ROM 芯片比规格上说的好一点，这台电脑从一开始就超频了。

更大的问题是如何与网络接口。对于非常旧的计算机来说，这是一个问题，但是 PPP 仍然存在，如果你有软件栈，你可以通过串行连接从服务器读取一些东西。[Alistar]实际上发现 UART 频率比 VGA 的点时钟频率更重要，因此系统时钟必须围绕串行端口而不是显示接口构建。这意味着文本模式界面实际上是 96 列，而不是通常的 80 列。

说你正在一块面包板上建造一台计算机是非常容易的。实际去做完全是另一回事。这实际上是一个令人惊讶的精心设计的计算机系统的草图，理论上，它将能够连接到互联网。当然，现实情况是，这台计算机将通过串行连接到一台连接到互联网的计算机，但这并不可耻。你可以在 GitHub 上查看这个项目的进度。
