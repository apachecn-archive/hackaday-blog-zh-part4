# 有什么区别？问运算放大器

> 原文:[https://hack aday . com/2018/11/13/what-the-difference-ask-an-op-amp/](https://hackaday.com/2018/11/13/whats-the-difference-ask-an-op-amp/)

如果你想知道为什么运算放大器上有小小的加号和减号，那是因为它的核心是一个差分放大器。问题是——至少在理想情况下——它具有无限增益，因此它像比较器一样工作，而这不是你通常想要的。因此，我们在它周围放置电阻来限制它，并从中获得有用的放大。[Stephen Mendes]不会为您分析差分放大器的标准配置如何工作。他假设你知道反相和同相放大器配置的常用公式，并使用叠加法。

[Stephen]提到这是最简单的方法，然后继续做，就像我们做检查一样。我们认为那是更容易的方法，但也许这是个人喜好的问题。不管怎样，你都会得到正确的答案。

有了叠加，你基本上一次忽略一个输入。当一个电源短接到地时，电路变成反相放大器，当另一个电源短接时，电路变成同相放大器。叠加告诉你，真正的答案是那两个答案之和。

另一种方法——我们不愿称之为更简单的方法——是记住关于运算放大器的两个简化假设。首先，假设运算放大器会尽一切努力使正负端具有相同的电压。接下来，假设输入完全开放，对于今天的运算放大器来说，这与事实相差不远。

基于此，很容易看出 V2 被 R3 和 R4 的分压器分压。所以你知道正电压。根据我们的假设，你也知道负电压，因为它应该是相同的。如果你知道电压和 R1，你可以计算通过 R1 电流。因为运算放大器的端子是开路的(或者至少我们是假装开路的),所以流经 R1 的电流一定也是流经 R2 的电流。所以我们知道 R2 两端的电压，从而知道 Vo 处的电压。

让我们试试这个的数字版本。假设 V1=5，V2=8。R1 和 R3=10K 和 R2 和 R4 是 20K。

*   Vplus 是 8V 乘以 R4/(R3+R4)或 5.3V
*   这意味着 Vminus 也是 5.3V
*   通过 R1 的电流必须是(5.3-5)/10000 = 33.3 微安(符号取决于你认为它流动的方式)
*   通过 R2 的电流一定是相同的，所以是 33.3 微安
*   为了在 R2 一端有 5.3 V 电压的情况下获得该电流，需要求解. 0000333 = (Vo-5.3)/20000
*   如果你携带足够多的数字，那就是 6V；如果你像我一样四舍五入，你会得到 5.966 伏，这足够接近了

如果你像[Stephen]那样计算，你会看到增益为 2，由于电压差为 3V，6V 输出非常合理。当然，假设只是在一定程度上是好的。轨电压为+/-15V 的运算放大器无法产生 50V 来将输入转换为特定值。但是对于常识性的应用来说，这些假设会让你走很长一段路。

这样容易还是那样容易？我们不能说。但叠加本身就是一个重要的概念，通常可以驯服一个难以驾驭的电路，所以即使你认为在这种特殊情况下很难做到这一点，仍然值得遵循数学。
